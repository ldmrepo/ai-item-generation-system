# 2.1.1 Curriculum 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: Curriculum
-   **설명**: “교육과정 전체”를 대표하는 노드로, 예를 들어 “2022 개정 수학과 교육과정” 같은 상위 개념을 저장.
    -   국가 수준 교육과정 또는 특정 교과(과목) 수준의 총괄 교육과정에 대한 정보를 담는다.

---

## 2. 필드(컬럼) 정의

| 필드명                | 자료형           | 제약조건         | 설명                                                        | 예시값                                                         |
| --------------------- | ---------------- | ---------------- | ----------------------------------------------------------- | -------------------------------------------------------------- |
| **id**                | 문자열(고유, PK) | UNIQUE, NOT NULL | Curriculum 노드의 고유 식별자 (PK)                          | `"CURR-2022-SMT"`                                              |
| **name**              | 문자열           | NOT NULL         | 교육과정 이름                                               | `"2022 개정 수학과 교육과정"`                                  |
| **year**              | 정수             |                  | 교육과정 개정 연도                                          | `2022`                                                         |
| **description**       | 문자열           |                  | 교육과정에 대한 개략적인 요약 설명                          | `"미래 사회 대비, 핵심역량 함양 중심의 수학과 교육과정"`       |
| **goals**             | 문자열 리스트    |                  | 교육과정 목표(복수)                                         | `["문제해결력 함양", "창의적 사고 증진"]`                      |
| **human_vision**      | 문자열           |                  | 교육과정이 지향하는 인간상(총론에서 제시하는 핵심 인재상)   | `"포용성과 창의성을 갖춘 주도적 사람"`                         |
| **basic_literacies**  | 문자열 리스트    |                  | 해당 교육과정에서 강조하는 기초 소양(수리, 디지털, 언어 등) | `["수리 소양", "디지털 소양", "언어 소양"]`                    |
| **subject_character** | 문자열           |                  | 수학 교과의 성격(학습 필요성·고유 특성 등)을 기술           | `"수학은 논리적 사고와 문제해결 역량을 기르는 핵심 교과이다."` |

### 필드별 상세 설명

1. **id (PK)**

    - 유형: 문자열
    - 제약: 고유(UNIQUE), NOT NULL
    - 예) `"CURR-2022-SMT"`, `"CURR-2022-MATH"`, 등.
    - 용도: Curriculum 노드를 식별하기 위한 유일한 값으로, 다른 노드·관계에서 참조할 때 사용.

2. **name**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"2022 개정 수학과 교육과정"`
    - 용도: 교육과정의 공식 명칭을 저장.

3. **year**

    - 유형: 정수
    - 예) `2022`
    - 용도: 교육과정의 개정 연도나 시행 연도 등. 필요 시 학년도 개념과 혼동되지 않도록 주의.

4. **description**

    - 유형: 문자열
    - 예) `"미래 사회 대비, 핵심역량 중심의 수학 교육과정 개정"`
    - 용도: 교육과정의 간략 개요나 특징을 자유롭게 서술.

5. **goals**

    - 유형: 문자열 리스트
    - 예) `["문제해결력 강화", "창의적 사고 함양", "협력적 소통 능력 증진"]`
    - 용도: 교육과정이 달성하고자 하는 주요 목표(복수)를 저장.

6. **human_vision**

    - 유형: 문자열
    - 예) `"포용성과 창의성을 갖춘 주도적 사람"`
    - 용도: 총론에서 제시하는 인간상(미래 인재상)을 저장.

7. **basic_literacies**

    - 유형: 문자열 리스트
    - 예) `["수리 소양", "디지털 소양", "언어 소양"]`
    - 용도: 총론 차원에서 강조하는 기초소양들을 저장. 교과 교육과정에 해당 소양이 어떻게 반영되는지 설명할 때 사용 가능.

8. **subject_character**
    - 유형: 문자열
    - 예) `"수학은 수리적 사고와 논리적 사고력을 향상시키는 교과로서, 문제해결 역량을 핵심으로 삼는다."`
    - 용도: 수학 교과 고유의 성격, 필요성, 학습 특성 등을 저장. (특정 교과(과목)에 대한 성격)

## 4. 제약(Constraints) 및 주의사항

-   **id**는 노드의 고유 식별자로써 중복 불가(UNIQUE).
-   **name**과 같은 필드는 NOT NULL을 권장.
-   **year**는 숫자 형식. (범위 제한은 정책에 따라 추가)
-   **goals**, **basic_literacies**는 여러 값을 담을 수 있는 리스트이므로, JSON 배열 등의 형태로 저장.
-   **subject_character**는 수학 교과 성격에 대한 구체적 서술. 만약 총론 차원의 문서라면 요약 형태로 기입.

## 5. 활용 시나리오

-   **학습 시스템**에서 “`CURR-2022-MATH`” 노드를 조회하여, 해당 교육과정이 지향하는 **핵심 목표**, **기초 소양**, **인간상**을 확인.
-   수학 교육과정(`Curriculum`) 노드와 하위 `Domain` 노드(예: “수와 연산”, “도형과 측정” 등)를 연결(`:HAS_DOMAIN`)해, 전체 구조를 파악.
-   다른 교과 교육과정(`Curriculum`) 노드와도 병렬로 관리 가능.

### 정리

**Curriculum** 노드는 “교육과정 전체”의 최상위 개념을 관리하는 핵심 노드입니다. `id`, `name`, `year`, `description`, `goals`, `human_vision`, `basic_literacies`, `subject_character` 필드를 통해 국가 수준(또는 시도/학교 수준)의 교육과정 정보를 일관성 있게 저장할 수 있습니다. Neo4j 상에서는 **(Curriculum)-[:HAS_DOMAIN]->(Domain)** 등으로 영역 노드와 연결함으로써, 교육과정 구조를 계층적으로 표현하게 됩니다.

# 2.1.2 Domain 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: Domain
-   **설명**: 교육과정에서 정의되는 ‘영역’을 나타내는 노드로, 예컨대 **수와 연산**, **변화와 관계**, **도형과 측정**, **자료와 가능성** 등 수학 교과의 상위 주제 영역을 저장한다.
-   주로 **Curriculum** 노드(예: “2022 개정 수학과 교육과정”)와 **Domain** 노드가 **“:HAS_DOMAIN”** 관계로 연결되어, 교육과정 구조를 계층적으로 표현한다.

---

## 2. 필드(컬럼) 정의

| 필드명          | 자료형           | 제약조건         | 설명                                     | 예시값                                                 |
| --------------- | ---------------- | ---------------- | ---------------------------------------- | ------------------------------------------------------ |
| **id**          | 문자열(고유, PK) | UNIQUE, NOT NULL | Domain 노드의 고유 식별자                | `"DMN-MATH-01"`                                        |
| **name**        | 문자열           | NOT NULL         | 영역 이름(“수와 연산”, “도형과 측정” 등) | `"도형과 측정"`                                        |
| **code**        | 문자열           |                  | 영역 코드를 간단히 표기                  | `"GEOM-MEAS"`                                          |
| **description** | 문자열           |                  | 해당 영역에 대한 요약·설명               | `"도형의 성질, 측정 단위, 넓이·부피 등을 다루는 영역"` |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 예) `"DMN-MATH-01"`, `"DMN-GEOM"` 등
    - 용도: Domain 노드를 식별하기 위한 유일한 값. 다른 노드(예: DomainAchievementLevel)나 관계에서 참조 가능.

2. **name**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"도형과 측정"`, `"자료와 가능성"`, `"수와 연산"`
    - 용도: 영역의 공식 혹은 간략한 명칭.

3. **code**

    - 유형: 문자열
    - 예) `"GEOM-MEAS"`, `"DATA-PROB"`
    - 용도: 영역을 코드 형태로 관리하려는 경우. 프로젝트에 따라 필수·선택 적용.

4. **description**
    - 유형: 문자열
    - 예) `"도형을 관찰·분석하고, 길이·넓이·부피 등 측정 개념을 학습하는 영역"`
    - 용도: 영역의 간략한 개요나 핵심 학습 내용을 요약.

---

## 3. 제약(Constraints)

-   **id**: 고유 식별자(UNIQUE), 필수(NOT NULL).
-   **name**: 필수(NOT NULL).
-   **code**, **description**: 선택적으로 입력 가능.

## 5. 활용 시나리오

-   **Curriculum** 노드(예: `"CURR-2022-MATH"`)와 **Domain** 노드가 **“:HAS_DOMAIN”** 관계로 연결됨.
    -   예) `(Curriculum {id: "CURR-2022-MATH"})-[:HAS_DOMAIN]->(Domain {id: "DMN-MATH-01"})`
-   상위 Curriculum에 속하는 여러 Domain(“수와 연산”, “도형과 측정”, “변화와 관계”, “자료와 가능성”)을 트리 형태로 표현해, **교육과정 구조**를 체계화.
-   각 Domain 노드와 **AchievementStandard** 노드가 연결됨(`:BELONGS_TO_DOMAIN`), “어느 영역에 속하는 성취기준인지” 표현 가능.

# 2.1.3 CoreIdea 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: CoreIdea
-   **설명**: 각 **영역(Domain)**을 관통하거나 대표하는 “핵심 아이디어”를 저장하는 노드
    -   예: “자연수와 정수 체계 확장”, “함수적 사고”, “확률·통계적 직관” 등 수학 교과의 주요 개념이나 가치
    -   주로 (Domain)-[:HAS_CORE_IDEA]->(CoreIdea) 관계로 연결됨

---

## 2. 필드(컬럼) 정의

| 필드명                    | 자료형           | 제약조건         | 설명                                                                   | 예시값                                                  |
| ------------------------- | ---------------- | ---------------- | ---------------------------------------------------------------------- | ------------------------------------------------------- |
| **id**                    | 문자열(고유, PK) | UNIQUE, NOT NULL | CoreIdea 노드의 고유 식별자 (PK)                                       | `"CI-001"`                                              |
| **content**               | 문자열           | NOT NULL         | 핵심 아이디어 내용(문장 혹은 키워드)                                   | `"수 체계 확장과 사칙연산의 일관성 이해"`               |
| **transferability_level** | 문자열           |                  | 전이 가능성 수준(“높음”, “중간”, “낮음” 등)                            | `"높음"`                                                |
| **vertical_aspect**       | 문자열           |                  | 수직적 확장(위계) 설명(예: 개념이 학년군을 올라가며 어떻게 확장되는지) | `"초등→음수, 중등→유리수, 고등→복소수로 확장되는 개념"` |
| **horizontal_aspect**     | 문자열           |                  | 수평적 연결 설명(예: 다른 영역이나 교과와의 연관성 등)                 | `"다른 영역(함수, 기하)에도 응용되는 연산 성질과 구조"` |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"CI-001"`, `"CORE-ALGEBRA"` 등
    - 용도: CoreIdea 노드를 식별하는 유일한 값.

2. **content**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"연산 개념의 통합적 이해"`, `"함수적 사고와 모형화"`
    - 용도: 핵심 아이디어의 핵심 문구. 비교적 간략하게 적되, 영역을 관통하는 아이디어를 표현.

3. **transferability_level**

    - 유형: 문자열
    - 예) `"높음"`, `"중간"`, `"낮음"`, 혹은 `"보통"`
    - 용도: 핵심 아이디어가 다른 주제·영역·학습 상황으로 얼마나 잘 전이되는지(활용도가 높은 개념인지)를 서술.

4. **vertical_aspect**

    - 유형: 문자열
    - 예) `"초등에서 정수, 중등에서 유리수, 고등에서 실수/복소수로 확장됨"`
    - 용도: 수직적 확장(학년이 올라갈수록 개념·원리가 어떻게 심화·발전되는지) 서술.

5. **horizontal_aspect**
    - 유형: 문자열
    - 예) `"다른 영역(함수, 기하) 및 타 교과(물리, 화학)에서의 연산 응용사례와 연계 가능"`
    - 용도: 수평적 연결(동일 학년군이나 다른 교과 등) 서술.

---

## 3. 제약(Constraints)

-   **id**: 고유 식별자, UNIQUE, NOT NULL.
-   **content**: 핵심 문구, NOT NULL.
-   **transferability_level**, **vertical_aspect**, **horizontal_aspect**: 필요 시 선택 입력.

    "vertical_aspect": "초등(자연수) → 중등(정수,유리수) → 고등(복소수)로 확장",
    "horizontal_aspect": "다른 영역(함수, 기하)에서도 연산 구조 활용 가능",

## 5. 활용 시나리오

-   **Domain** 노드(예: `"도형과 측정"`)와 **CoreIdea** 노드(예: `"CI-003" - '도형의 변환과 합동·닮음 이해'"`)가 “:HAS_CORE_IDEA” 관계로 연결 → 영역별 핵심 아이디어를 구조화.
-   **vertical_aspect**와 **horizontal_aspect**를 활용하여, 학년군 간 내용의 연계(수직적), 타 영역·타 교과와의 융합 가능성(수평적)을 DB에서 조회·분석할 수 있음.

### 요약

**CoreIdea** 노드는 수학 교과 각 **영역**(Domain)에서 관통하는 “핵심 아이디어”를 표현합니다. `id`로 고유 식별을 하고, `content`에 핵심 개념을 간결히 기록합니다. 이어서 `transferability_level`, `vertical_aspect`, `horizontal_aspect` 등을 통해 개념의 전이 가능성, 학년군별 확장(수직), 다른 영역·과목과의 연계(수평)를 저장함으로써, 수학 학습 전반에서 아이디어의 구조와 활용도를 체계적으로 관리할 수 있습니다.

1. **id**

4. **school_level**
    - 유형: 문자열
    - 예) `"초등학교"`, `"중학교"`, `"고등학교"`
    - 용도: 어느 학교급에 해당하는 학년군인지. 예: 초등 1~2학년군은 school_level=“초등학교”.

-   **order**, **school_level**: 필요 시 선택 입력.

# 2.1.5 ContentCategory 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: ContentCategory
-   **설명**: 특정 영역(Domain) 내에서 더 세분화된 내용 범주를 저장하는 노드. 예를 들어 **“수와 연산” 영역** 내에서 “정수와 유리수”, “분수·소수”, “사칙연산 확장” 등 하위 범주를 만들거나, **“도형과 측정”** 내에서 “삼각형과 사각형”, “도형의 변환”, “측정(길이·넓이·부피)” 등을 분류하는 식으로 사용할 수 있음.
-   이 노드를 통해 **한 영역(Domain)을 더 세밀히 구조화**하여, 성취기준이나 내용 요소를 체계적으로 관리할 수 있다.

---

## 2. 필드(컬럼) 정의

| 필드명          | 자료형           | 제약조건         | 설명                                              | 예시값                                        |
| --------------- | ---------------- | ---------------- | ------------------------------------------------- | --------------------------------------------- |
| **id**          | 문자열(고유, PK) | UNIQUE, NOT NULL | ContentCategory 노드의 고유 식별자                | `"CC-ARITH-01"`                               |
| **name**        | 문자열           | NOT NULL         | 범주 이름(“정수와 유리수”, “삼각형과 사각형”, 등) | `"정수와 유리수"`                             |
| **description** | 문자열           |                  | 해당 내용 범주에 대한 설명                        | `"정수·유리수의 대소 관계, 사칙연산 확장 등"` |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 예) `"CC-ARITH-01"`, `"CC-GEOM-01"`, `"CAT-002"` 등
    - 용도: ContentCategory 노드를 식별하는 고유 값. 다른 노드나 관계에서 참조 가능.

2. **name**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"정수와 유리수"`, `"삼각형과 사각형"`, `"확률 개념"`
    - 용도: 실제 현장에서 사용하는 범주의 명칭. 예: “1. 정수와 유리수” 또는 “도형 변환”

3. **description**
    - 유형: 문자열
    - 예) `"정수와 유리수의 개념, 사칙연산, 대소 비교 등을 다룸."`
    - 용도: 범주에 대한 간략한 소개나 개념적 범위 설명.

---

## 3. 제약(Constraints)

-   **id**: 고유 식별자(UNIQUE), NOT NULL.
-   **name**: NOT NULL(범주 이름 필수).
-   **description**: 선택적으로 입력 가능.

    "description": "정수와 유리수의 개념, 대소 관계, 사칙연산 확장 등을 다루는 범주",

## 5. 활용 시나리오

-   **Domain** 노드(예: “수와 연산”)와 **ContentCategory** 노드(예: “정수와 유리수”)가 “:HAS_CATEGORY” 관계로 연결됨.
    -   예: `(Domain {id:"DMN-MATH-01"})-[:HAS_CATEGORY]->(ContentCategory {id:"CC-ARITH-01"})`
-   ContentCategory 노드와 **ContentElement** 노드를 연결하여, 범주 내 세부 내용 요소를 구조화할 수도 있음.
    -   예: `(ContentCategory)-[:HAS_ELEMENT]->(ContentElement)`

이처럼 **ContentCategory** 노드를 통해 한 영역(Domain)을 더 세밀하게 구분·분류하여, **성취기준**이나 **내용 요소(ContentElement)**와 체계적으로 연결함으로써, 교육과정 내용을 직관적으로 관리·검색할 수 있다.

---

### 요약

**ContentCategory**는 **“영역(Domain)”**을 더 세밀하게 분류한 내용 범주를 나타내며, `id`, `name`, `description`을 핵심 필드로 사용합니다. 이를 통해 **도메인(영역)** 내에 여러 개의 학습 범주를 체계적으로 저장·연결하여, **성취기준**이나 **내용 요소**를 한층 더 구체적으로 조직할 수 있습니다.

# 2.1.6 ContentElement 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: ContentElement
-   **설명**: **내용 요소**를 나타내는 노드로, 교육과정에서 특정 내용 범주(ContentCategory) 내에 속하는 구체적인 개념 · 원리 · 법칙 등을 저장.
    -   예: “분수의 곱셈과 나눗셈”, “이등변삼각형 작도”, “일차함수와 그래프 해석” 등 구체 학습 요소
    -   주로 (ContentCategory)-[:HAS_ELEMENT]->(ContentElement) 등의 관계로 연결됨

---

## 2. 필드(컬럼) 정의

| 필드명                 | 자료형           | 제약조건         | 설명                                                                           | 예시값                   |
| ---------------------- | ---------------- | ---------------- | ------------------------------------------------------------------------------ | ------------------------ |
| **id**                 | 문자열(고유, PK) | UNIQUE, NOT NULL | ContentElement 노드의 고유 식별자                                              | `"CE-ARITH-05"`          |
| **content**            | 문자열           | NOT NULL         | 구체적인 내용 요소의 이름 또는 설명                                            | `"분수의 곱셈과 나눗셈"` |
| **category**           | 문자열           |                  | 해당 내용 요소가 속하는 범주 코드(ContentCategory.id 등)                       | `"CC-ARITH-01"`          |
| **hierarchical_level** | 정수             |                  | 내용 요소의 위계 수준(1=기초, 2=중간, 3=심화 등) (선택적)                      | `1`                      |
| **domain_id**          | 문자열           |                  | 소속 영역(Domain)의 id. (필요 시 직접 참조 가능, 또는 Category 통해 간접 참조) | `"DMN-MATH-01"`          |
| **grade_group_id**     | 문자열           |                  | 주로 다루는 학년군(GradeGroup)의 id. (선택, 혹은 Category 통해 간접 매핑)      | `"GG-EL34"`              |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 예) `"CE-ARITH-05"`, `"CE-GEOM-02"` 등
    - 용도: ContentElement 노드를 식별하는 유일한 값.

2. **content**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"분수의 곱셈과 나눗셈"`, `"삼각형 작도"`, `"함수의 그래프 해석"`
    - 용도: 내용 요소를 구체적으로 나타내는 명칭·주제.

3. **category**

    - 유형: 문자열
    - 예) `"CC-ARITH-01"`, `"CC-GEOM-03"`
    - 용도: 해당 ContentElement가 어느 ContentCategory에 속하는지를 나타내는 code/id.
    - 만약 (ContentCategory)-[:HAS_ELEMENT]->(ContentElement) 관계가 있다면, 중복 정보가 될 수 있으나, DB 질의 편의상 필드를 유지할 수도 있음.

4. **hierarchical_level**

    - 유형: 정수
    - 예) `1`(기초), `2`(중간), `3`(심화) 등
    - 용도: 해당 내용 요소가 어느 수준(위계)에 해당하는지. 구체적인 체계는 교육과정 정책에 따라 결정.

5. **domain_id**

    - 유형: 문자열
    - 예) `"DMN-MATH-01"`
    - 용도: 직접 영역(Domain) id를 기록하여 참조할 수 있음. (또는 Category 노드를 통해 간접적으로 알 수 있음)

6. **grade_group_id**
    - 유형: 문자열
    - 예) `"GG-EL34"`, `"GG-MS13"`
    - 용도: 이 내용 요소가 주로 다루이는 학년군을 표시(직접 연결). (Category나 Domain을 통해 간접 파악 가능하나, 필요 시 중복해서 저장)

## 3. 제약(Constraints)

-   **id**: UNIQUE, NOT NULL
-   **content**: NOT NULL
-   **category**, **domain_id**, **grade_group_id**: 필요 시 NULL 가능.
-   **hierarchical_level**: 정수 범위 정책(1~3, 1~N 등) 임의 설정 가능.

-   (Curriculum)-[:HAS_COURSE]->(CourseStructure) 등의 관계로, “2022 개정 수학과 교육과정”이 어떤 과목 구조(공통수학1/2, 일반선택 대수·미적분I 등, 진로선택 미적분II·기하·경제수학 등, 융합선택 과목 등)을 포함하는지 명시.

### 요약

**CourseStructure** 노드는 고등학교 수학 과목 구조(공통·일반선택·진로선택·융합선택 등)와 구체적 과목명(“공통수학1”, “미적분I” 등)을 DB에 표현합니다. `id`, `name`이 핵심 필드이며, `description`에 과목 범위나 성격을 추가 설명합니다. 이를 통해 **Curriculum** 노드와 연결하여 고등학교 수학의 과목 편성 체계를 체계적으로 관리·조회할 수 있습니다.

# 2.1.10 EducationalElement 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: EducationalElement
-   **설명**: **교수·학습 환경 요소**(예: 생태전환 교육, 민주 시민 교육, 안전·건강, 다문화, 독도 교육, 등)를 관리하기 위한 노드.
    -   “생태전환교육”, “민주시민교육”, “인성교육”, “진로연계교육” 등과 같이 총론에서 강조하는 범교과 학습 주제나, 특정 교수·학습 중점을 나타내는 요소도 저장할 수 있음.
-   교육과정(특히 총론이나 개정 중점)에서 강조하는 특별 주제·환경 요소를 DB에서 구조적으로 나타낼 때 사용.

---

## 2. 필드(컬럼) 정의

| 필드명          | 자료형           | 제약조건         | 설명                                                    | 예시값                                             |
| --------------- | ---------------- | ---------------- | ------------------------------------------------------- | -------------------------------------------------- |
| **id**          | 문자열(고유, PK) | UNIQUE, NOT NULL | EducationalElement 노드의 고유 식별자                   | `"EE-ECO"`                                         |
| **name**        | 문자열           | NOT NULL         | 요소 이름(“생태전환교육”, “민주시민교육” 등)            | `"생태전환교육"`                                   |
| **description** | 문자열           |                  | 해당 교수·학습 환경 요소에 대한 설명                    | `"환경·생태계 보존을 위한 학습 활동 및 실천 강조"` |
| **focus_area**  | 문자열           |                  | 적용 영역·중점 (“안전교육”, “민주 시민”, “생태환경” 등) | `"생태환경"`                                       |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"EE-ECO"`, `"EE-DEMO"`, `"EE-MULTI"` 등
    - 용도: EducationalElement 노드를 식별하는 고유값. 다른 노드나 관계에서 참조 시 사용.

2. **name**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"생태전환교육"`, `"민주시민교육"`, `"안전·건강교육"`
    - 용도: 교과(총론)에서 강조하는 환경 요소·주제 이름.

3. **description**

    - 유형: 문자열
    - 예) `"학습 전반에서 환경·생태 보전을 실천하도록 유도하는 생태전환 교육"`
    - 용도: 해당 요소의 구체적 의미나 특징을 설명.

4. **focus_area**
    - 유형: 문자열
    - 예) `"환경"`, `"민주 시민"`, `"다문화"`, `"인성"`
    - 용도: 이 요소가 주로 다루는 범주나 중점 분야를 한 단어 혹은 짧은 문장으로.
    - 선택 항목으로, 필요 시 “환경”, “민주 시민”, “인권·다문화” 등 분류 가능.

---

## 3. 제약(Constraints)

-   **id**: UNIQUE, NOT NULL (고유 식별자)
-   **name**: NOT NULL (교수·학습 환경 요소명)
-   **description**, **focus_area**: 필요 시 선택 입력

# 2.2.1 AchievementStandard 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: AchievementStandard
-   **설명**: 교육과정에서 제시하는 **성취기준**을 저장하는 노드. 예) “[2수01-06] 두 자리 수의 덧셈과 뺄셈”, “[9수02-04] 일차방정식을 풀고 활용하기” 등.
    -   성취기준은 특정 학년군(GradeGroup)·영역(Domain)에서 학습자가 도달해야 할 지식 · 이해, 과정 · 기능, 가치 · 태도를 구체적으로 기술하는 교육과정 문항이다.

---

## 2. 필드(컬럼) 정의

| 필드명                             | 자료형           | 제약조건         | 설명                                                                   | 예시값                                                                     |
| ---------------------------------- | ---------------- | ---------------- | ---------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| **id**                             | 문자열(고유, PK) | UNIQUE, NOT NULL | AchievementStandard 노드의 고유 식별자                                 | `"AS-2수01-06"`                                                            |
| **content**                        | 문자열           | NOT NULL         | 성취기준 문구                                                          | `"[2수01-06] 두 자리 수 덧셈·뺄셈의 계산 원리를 이해하고 계산할 수 있다."` |
| **learningObjective**              | 문자열           |                  | 해당 성취기준을 통해 달성하고자 하는 학습 목표(요약)                   | `"두 자리 수 범위에서 덧셈·뺄셈의 계산 방법을 익혀 문제를 해결한다."`      |
| **difficulty_level**               | 문자열           |                  | 난이도 수준(“낮음”, “중간”, “높음” 등)                                 | `"중간"`                                                                   |
| **detail_description**             | 문자열           |                  | 성취기준에 대한 보충 설명                                              | `"덧셈과 뺄셈의 관계를 이해하고, 세 수 이상 혼합 계산에도 적용 가능."`     |
| **assessment_elements**            | 문자열 리스트    |                  | 해당 성취기준을 평가할 때 중점적으로 확인할 요소(복수)                 | `["계산 정확도", "연산 감각", "문제 해결 과정"]`                           |
| **domain_id**                      | 문자열           |                  | 어느 영역(Domain)에 속하는지(“DMN-MATH-01” 등)                         | `"DMN-MATH-01"`                                                            |
| **topic**                          | 문자열           |                  | 성취기준이 다루는 주제(“도형 그리기”, “분수 사칙연산”, 등)             | `"두 자리 수 연산"`                                                        |
| **knowledge_component**            | 문자열           |                  | 지식·이해 측면에서 이 성취기준의 핵심 요소                             | `"두 자리 수의 자릿값 이해, 덧셈·뺄셈 계산 원리"`                          |
| **skill_component**                | 문자열           |                  | 과정·기능 측면에서 이 성취기준의 핵심 요소                             | `"세 수 혼합 계산, 계산 결과 어림하기, 반성하기"`                          |
| **value_attitude_component**       | 문자열           |                  | 가치·태도 측면에서 이 성취기준의 핵심 요소                             | `"계산 정확성을 바탕으로 한 자신감, 협력적 문제 해결 태도"`                |
| **grade_group**                    | 문자열           |                  | 어느 학년군(GradeGroup)에 해당하는지(“GG-EL12” 등)                     | `"GG-EL12"`                                                                |
| **standard_explanation**           | 문자열           |                  | (가) 성취기준 해설 부분. 문서 본문에서 각 성취기준 해설을 옮길 수 있음 | `"덧셈은 세 자리 수의 합이 네 자리 수를 초과해도 계산 원리를 유지한다."`   |
| **considerations_for_application** | 문자열           |                  | (나) 성취기준 적용 시 고려 사항                                        | `"난이도가 높은 혼합 계산은 공학 도구 사용을 허용할 수 있다."`             |
| **minimumRequiredLevel**           | 문자열           |                  | (선택) “C” 등 최소 성취수준을 지정할 때 사용                           | `"C"`                                                                      |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 예) `"AS-2수01-06"`, `"AS-9수02-04"`, `"AS-HS-COMMON1-01"` 등
    - 용도: AchievementStandard 노드를 식별하는 유일한 값.
    - 다른 노드(예: AchievementLevelStandard)에서 참조할 때 사용.

2. **content**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"[2수01-06] 두 자리 수 범위에서 덧셈과 뺄셈을 계산할 수 있다."`
    - 용도: 실제 성취기준 문구 원문.

3. **learningObjective**

    - 유형: 문자열
    - 예) `"두 자리 수 범위의 덧셈·뺄셈 계산 방법을 습득하여 일상 문제를 해결한다."`
    - 용도: 성취기준의 학습 목표 요약. (선택 필드)

4. **difficulty_level**

    - 유형: 문자열
    - 예) `"낮음"`, `"중간"`, `"높음"`
    - 용도: 성취기준 난이도를 추정하여 분류.

5. **detail_description**

    - 유형: 문자열
    - 예) `"두 자리 수의 자릿값 개념을 심화하고, 받아올림·내림 등을 다룬다."`
    - 용도: 성취기준에 대한 추가 해설. (수준별 지도 시 참고)

6. **assessment_elements**

    - 유형: 문자열 리스트
    - 예) `["계산 정확도", "문제 해결 전략", "연산 감각"]`
    - 용도: 평가시 확인해야 할 요소(“무엇을 보여주면 성취기준 도달로 볼 수 있는가”).

7. **domain_id**

    - 유형: 문자열
    - 예) `"DMN-MATH-01"` (수와 연산)
    - 용도: 어느 영역(Domain)에 속하는 성취기준인지 표시.
    - (AchievementStandard)-[:BELONGS_TO_DOMAIN]->(Domain)의 관계로도 표현할 수 있음.

8. **topic**

    - 유형: 문자열
    - 예) `"두 자리 수 연산"`, `"분수 사칙연산"`, `"이차방정식 풀이"`, 등
    - 용도: 성취기준이 다루는 세부 주제를 추가로 나타낼 수 있음.

9. **knowledge_component**

    - 유형: 문자열
    - 예) `"두 자리 수 자릿값, 덧셈·뺄셈의 원리 이해"`
    - 용도: 지식·이해 관점에서 핵심 내용을 요약.

10. **skill_component**

    - 유형: 문자열
    - 예) `"세 수 혼합 계산, 받아올림/내림 절차 숙달"`
    - 용도: 과정·기능 관점에서 핵심 스킬·절차를 명시.

11. **value_attitude_component**

    - 유형: 문자열
    - 예) `"계산 과정을 스스로 반성하고, 협력 활동에서 서로의 방법 존중"`
    - 용도: 가치·태도 관점에서 중점적으로 길러야 할 태도.

12. **grade_group**

    - 유형: 문자열
    - 예) `"GG-EL12"`, `"GG-MS13"`, `"GG-HIGH"`
    - 용도: 어느 학년군에 속하는 성취기준인지 명시.
    - (AchievementStandard)-[:FOR_GRADE_GROUP]->(GradeGroup) 관계와 중복이 될 수 있으나 편의상 필드 유지 가능.

13. **standard_explanation**

    - 유형: 문자열
    - 예) `"이 성취기준 해설: 덧셈과 뺄셈은 역연산 관계로, 한 가지 상황을 여러 식으로 해석..."`
    - 용도: 교육과정 문서에서 “(가) 성취기준 해설” 부분을 그대로 저장할 수 있음.

14. **considerations_for_application**

    - 유형: 문자열
    - 예) `"세 수 이상 혼합 계산은 학습 부담이 클 수 있으므로, 공학도구 사용 가능."`
    - 용도: 교육과정 문서에서 “(나) 성취기준 적용 시 고려사항”을 저장.

15. **minimumRequiredLevel**
    - 유형: 문자열
    - 예) `"C"`
    - 용도: (선택) 이 성취기준에서 최소 도달해야 할 수준(“C” 등)을 지정할 때 사용.

---

## 3. 제약(Constraints)

-   **id**: UNIQUE, NOT NULL (PK)

# 2.2.2 AchievementLevel 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: AchievementLevel
-   **설명**:  
    교육과정에서 정의하는 **성취수준**(예: A/B/C, 또는 A/B/C/D/E)을 저장하는 노드.
    -   3수준 체계의 경우: A, B, C
    -   5수준 체계의 경우: A, B, C, D, E
    -   각 수준별로 설정된 점수 범위(예: “≥80%”), 고정 분할점수(예: 80점), 수준 체계(3단계 or 5단계) 등을 관리.
    -   **AchievementLevelStandard**와 연결해, 각 성취기준에 대한 구체적 수준(A/B/C…)을 표현 가능.

---

## 2. 필드(컬럼) 정의

| 필드명                 | 자료형           | 제약조건         | 설명                                                      | 예시값                     |
| ---------------------- | ---------------- | ---------------- | --------------------------------------------------------- | -------------------------- |
| **id**                 | 문자열(고유, PK) | UNIQUE, NOT NULL | AchievementLevel 노드의 고유 식별자                       | `"LEVEL-3A"`, `"LEVEL-5C"` |
| **name**               | 문자열           | NOT NULL         | 수준 이름(“A”, “B”, “C”, “D”, “E” 등)                     | `"A"`                      |
| **level_order**        | 정수             |                  | 수준 순서(3수준 체계 → 1~3, 5수준 체계 → 1~5 등)          | `1`                        |
| **level_system**       | 문자열           |                  | 어느 수준 체계인지(“3단계”, “5단계” 등)                   | `"3단계"`, `"5단계"`       |
| **description**        | 문자열           |                  | 수준 전반에 대한 설명(예: “3수준 A≥80%” or “5수준 A≥90%”) | `"3단계 A수준(≥80%)"`      |
| **score_range**        | 문자열           |                  | 점수 범위(“≥80%”, “70%~80%”) 등                           | `"≥80%"`                   |
| **fixed_cutoff_score** | 정수             |                  | 고정 분할점수(예: 중학교에서 80점 기준)                   | `80`                       |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"LEVEL-3A"`, `"LEVEL-5C"`, `"3LVL-A"`, `"5LVL-E"`
    - 용도: AchievementLevel 노드를 식별하는 유일한 값.

2. **name**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"A"`, `"B"`, `"C"`, `"D"`, `"E"`
    - 용도: 실제 수준 이름(알파벳).

3. **level_order**

    - 유형: 정수
    - 예) `1`, `2`, `3`, …
    - 용도: 성취수준을 순서대로 관리. 예: A(1), B(2), C(3) 등.

4. **level_system**

    - 유형: 문자열
    - 예) `"3단계"`, `"5단계"`
    - 용도: 어느 수준 체계인지 구분.

5. **description**

    - 유형: 문자열
    - 예) `"3수준 체계에서 A수준(≥80%)"`, `"5수준 체계 D수준(≥60%~70%)"`
    - 용도: 해당 수준에 대한 전반적인 설명.

6. **score_range**

    - 유형: 문자열
    - 예) `"≥80%"`, `"70%~80%"`, `"≥40%~60%"`
    - 용도: 점수 또는 성취율 구간을 표현.

7. **fixed_cutoff_score**
    - 유형: 정수
    - 예) `80`
    - 용도: 중학교 등에서 고정 분할점수(예: 80점)를 사용하여 성취수준을 결정할 때 활용. 필요 시 선택적으로 사용.

## 3. 제약(Constraints)

-   **id**: UNIQUE, NOT NULL(고유 식별자)
-   **name**: NOT NULL(“A/B/C” 등 수준 이름)
-   **level_order**, **level_system**, **description**, **score_range**, **fixed_cutoff_score**: 필요 시 선택 입력

## 5. 활용 시나리오

-   **AchievementLevel** 노드는 (AchievementLevelStandard)-[:AT_LEVEL]->(AchievementLevel) 관계를 통해, 각 성취기준별 성취수준(A/B/C… 등)을 매핑.
-   한 교육과정(또는 학교)에서 3수준 체계를 도입 시 “3LVL-A/B/C”만 저장.
-   다른 경우 5단계 체계를 도입 시 “5LVL-A/B/C/D/E” 저장.
-   점수 범위를 `score_range`에 기록해, 성취수준에 따른 **% or 점수 기준**을 DB에서 일관성 있게 관리.

---

### 요약

**AchievementLevel**는 **“성취수준 (A/B/C/D/E)”** 정보를 정의하는 노드로,

-   3수준(“3단계”)일지, 5수준(“5단계”)일지 `level_system`을 통해 구분
-   `name`(A,B,C,D,E)와 `level_order`(1,2,3,4,5), `score_range`(“≥80%” 등)로 수준을 체계화
-   `fixed_cutoff_score`로 고정 분할점수(예: 80점 기준)도 기록 가능

이렇게 DB에 성취수준 체계를 명확히 저장해 두면, **AchievementLevelStandard**와 연결하여 각 성취기준별로 “A/B/C…” 수준 정의를 쉽게 참조하고 관리할 수 있습니다.

    - 유형: 문자열
    - 예) `"가치·태도 내면화, 실천 범위 매우 넓음"`
    - 용도: 가치·태도 측면에서의 특징.

    - 유형: 부울
    - 예) `true` / `false`
    - 용도: 이 수준이 해당 성취기준의 **최소 성취수준**(보장해야 하는 수준)인지 여부.

-   **id**: UNIQUE, NOT NULL (고유 식별자)
-   **standard_id**: 필요 시 NOT NULL로 설정 (반드시 특정 성취기준과 연관됨)

# 2.2.4 DomainAchievementLevel 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: DomainAchievementLevel
-   **설명**: 특정 **영역(Domain)** 전체 관점에서 A/B/C(혹은 A~E)와 같은 성취수준을 정의하는 노드.
    -   예: “수와 연산(영역) 전체를 A/B/C로 나누어 성취수준을 나타낼 때, A수준은 어떠한 능력을 의미하는지, B수준은 어떠한 능력을 의미하는지”를 기록.
    -   일반적으로 **“영역 전체”**에서 성취수준별로 어느 정도 성취도(지식, 기능, 태도)를 갖추었는지 설명할 수 있으며, **common_misconceptions**(오개념), **teaching_suggestions** 등을 함께 기술해 두어 교수·학습 개선 자료로 활용한다.

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"DAL-Suyeon-A"`, `"DAL-Geom-B"`, `"DAL-Data-E"` 등
    - 용도: DomainAchievementLevel 노드를 식별하는 유일한 값.

2. **domain_id**

    - 유형: 문자열
    - 예) `"DMN-MATH-01"`(수와 연산 영역)
    - 용도: 어느 영역(Domain)에 대한 성취수준인지 표시.
    - 필요 시 (DomainAchievementLevel)-[:FOR_DOMAIN]->(Domain) 관계로도 표현 가능.

3. **level**

    - 유형: 문자열
    - 예) `"A"`, `"B"`, `"C"`, `"D"`, `"E"`
    - 용도: 3단계 or 5단계 체계에서 특정 수준을 말함.

4. **category**

    - 유형: 문자열
    - 예) `"CC-ARITH-01"`, `"CC-GEOM-02"`
    - 용도: (선택) 영역 내 특정 내용 범주에 국한되는 성취수준 설명을 기록할 때. 아니면 영역 전반을 의미.

5. **content**

    - 유형: 문자열
    - 예) `"이 수준(A)에서는 수 개념과 사칙연산을 다양한 문제에서 능숙하게 적용할 수 있음"`
    - 용도: 영역 전체 수준에서 학생이 전반적으로 보이는 성취 모습을 요약.

6. **grade_group**

    - 유형: 문자열
    - 예) `"GG-EL12"`, `"GG-MS13"`, `"GG-HIGH"`
    - 용도: (선택) 해당 영역 성취수준이 어느 학년군에 해당하는지 명시.

7. **competency_focus**

    - 유형: 문자열 리스트
    - 예) `["문제해결", "추론"]`
    - 용도: 이 수준에서 특히 중점이 되는 수학 교과 역량(문제해결, 추론, 정보처리 등).

8. **key_concepts**

    - 유형: 문자열 리스트
    - 예) `["분수 개념", "소인수분해", "최대공약수"]`
    - 용도: 영역에서 A수준 등급을 판단하는 데 핵심이 되는 개념·주제.

9. **prerequisite_knowledge**

    - 유형: 문자열
    - 예) `"초등 3~4학년 분수 개념"`
    - 용도: 이 수준 성취를 위해 반드시 갖춰야 할 선수 지식을 기술.

10. **common_misconceptions**

    - 유형: 문자열 리스트
    - 예) `["분수 곱셈에서 분모끼리도 곱해야 한다는 착각", "최소공배수 구하는 과정 오류"]`
    - 용도: 이 수준 학습 시 학생들이 흔히 범하는 오개념 목록.

11. **teaching_suggestions**

    - 유형: 문자열
    - 예) `"교구 활용한 분수 곱셈 시연, 학생 간 협력 과제 제시, 오개념 바로잡기 자료 제공"`
    - 용도: 교사가 수업을 설계할 때 참고할 지도 방안.

12. **assessment_focus**
    - 유형: 문자열
    - 예) `"복합연산 문제 해결, 분수 개념 정확도, 학습 태도"`
    - 용도: 평가할 때 중점적으로 확인할 요소.

-   다른 필드(content, category, grade_group, 등)는 선택 입력 가능.

## 2. 필드(컬럼) 정의

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"COMP-PROBLEM-SOLVE"`, `"COMP-COMM"`, `"COMP-CONNECT"` 등
    - 용도: Competency 노드를 식별하기 위한 유일한 값.

    - 제약: NOT NULL
    - 예) `"문제해결 역량"`, `"추론 역량"`, `"연결 역량"`, `"정보처리 역량"`
    - 용도: 교과 역량의 이름.

3. **description**

    - 유형: 문자열
    - 예) `"수학적 문제 상황을 해결하고 과정을 점검·반성할 수 있는 능력"`
    - 용도: 역량의 구체적 정의나 특성을 간략히 서술.

4. **core_elements**
    - 유형: 문자열 리스트
    - 예) `["문제 이해", "계획 수립", "전략 적용", "결과 반성"]`
    - 용도: 역량을 구성하는 핵심 요소들(복수)을 나타냄. 필요에 따라 사용.

---

## 3. 제약(Constraints)

-   **id**: UNIQUE, NOT NULL (고유 식별자)
-   **name**: NOT NULL (역량 이름)
-   **description**, **core_elements**: 선택 입력 가능

---

## 4. 예시 레코드

**예시 1**

```yaml
{
    "id": "COMP-PROBLEM-SOLVE",
    "name": "문제해결 역량",
    "description": "수학적 문제를 이해하고 해결 전략을 수립·실행하며 반성·평가할 수 있는 능력",
    "core_elements": ["문제 이해", "전략 탐색", "해결 과정 반성"],
}
```

**예시 2**

```yaml
{
    "id": "COMP-REASONING",
    "name": "추론 역량",
    "description": "관찰, 실험, 귀납·연역적 추론 등을 통해 수학적 사실을 도출하고 정당화하는 능력",
    "core_elements": ["관찰·실험", "귀납 추론", "정당화·증명"],
}
```

---

## 5. 활용 시나리오

-   **Curriculum** 노드(예: `"CURR-2022-MATH"`)와 Competency 노드(예: `"COMP-PROBLEM-SOLVE"`)가 “:HAS_COMPETENCY” 관계로 연결됨.
    -   즉, “2022 개정 수학과 교육과정”이 요구하는 교과 역량 중 하나로 “문제해결 역량”을 포함.
-   “(AchievementStandard)-[:DEVELOPS_COMPETENCY]->(Competency)” 관계로, 특정 성취기준이 어떤 역량을 중점적으로 다루는지 나타낼 수도 있음.
-   DB 질의 시, 사용자는 “이 교육과정에 포함된 교과 역량 목록” 조회 가능, 또는 “문제해결 역량을 강조하는 성취기준” 목록을 찾을 수 있음.

---

### 요약

**Competency** 노드는 **“수학 교과 역량”**(문제해결, 추론, 의사소통, 연결, 정보처리 등)을 저장하며, `id`(고유 식별자)와 `name`(역량 이름)이 핵심 필드입니다. `description`으로 역량 정의를, `core_elements`로 역량의 세부 요소를 표현합니다. 이를 **Curriculum**과 연결하면, 교육과정에서 강조하는 교과 역량을 DB상에서 일관성 있게 관리·활용할 수 있습니다.

아래는 **데이터 스키마**에서 정의된 **2.3.2 CompetencySubElement** 노드에 대한 **데이터 설명서**(Data Dictionary) 예시입니다. 노드의 목적, 필드(컬럼), 타입, 제약조건, 필드 설명, 그리고 예시값 등을 종합하여 작성하였습니다.

---

# 2.3.2 CompetencySubElement 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: CompetencySubElement
-   **설명**: 수학 교과 역량(Competency)의 하위 요소를 저장하는 노드.
    -   예: “문제해결 역량”을 좀 더 세분화하여 “문제 이해”, “전략 탐색”, “해결 과정 반성” 등 구체 항목으로 표현할 수 있음.
    -   주로 (CompetencySubElement)-[:BELONGS_TO_COMPETENCY]->(Competency) 관계를 통해 상위 역량과 연결.

---

## 2. 필드(컬럼) 정의

| 필드명                     | 자료형           | 제약조건         | 설명                                              | 예시값                                        |
| -------------------------- | ---------------- | ---------------- | ------------------------------------------------- | --------------------------------------------- |
| **id**                     | 문자열(고유, PK) | UNIQUE, NOT NULL | CompetencySubElement 노드의 고유 식별자           | `"CSE-PROBLEM-UNDERSTAND"`                    |
| **competencyId**           | 문자열           |                  | 어느 역량(Competency)과 연결되는지(Competency.id) | `"COMP-PROBLEM-SOLVE"`                        |
| **content**                | 문자열           | NOT NULL         | 하위 요소의 구체적 내용                           | `"문제 상황을 이해하고 핵심을 파악하는 능력"` |
| **development_strategies** | 문자열 리스트    |                  | 해당 하위 요소를 기르기 위한 교수·학습 전략(복수) | `["실생활 예시 활용", "질문 유도"]`           |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"CSE-PROBLEM-UNDERSTAND"`, `"CSE-REASONING-INDUCTIVE"` 등
    - 용도: CompetencySubElement 노드를 식별하는 유일한 값.

2. **competencyId**

    - 유형: 문자열
    - 예) `"COMP-PROBLEM-SOLVE"`, `"COMP-REASONING"`
    - 용도: 어느 상위 역량(Competency.id)에 속하는 하위 요소인지 연결.
    - 필요 시 (CompetencySubElement)-[:BELONGS_TO_COMPETENCY]->(Competency) 관계로 표현 가능.

3. **content**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"문제 상황 이해"`, `"가설 설정 및 귀납적 추론"`
    - 용도: 하위 요소의 이름·내용 요약.

4. **development_strategies**
    - 유형: 문자열 리스트
    - 예) `["실생활 문제 제시", "학생 주도 토론", "오개념 분석 후 피드백"]`
    - 용도: 이 하위 요소를 개발·향상하기 위한 교수·학습 전략(복수). 참고용.

---

## 3. 제약(Constraints)

-   **id**: UNIQUE, NOT NULL
-   **content**: NOT NULL
-   **competencyId**, **development_strategies**: 선택적 사용 가능.
-   상위 역량(Competency)와 1:N 관계를 형성할 수 있음.

---

## 4. 예시 레코드

**예시 1**

```yaml
{
    "id": "CSE-PROBLEM-UNDERSTAND",
    "competencyId": "COMP-PROBLEM-SOLVE",
    "content": "문제 상황을 이해하고, 핵심 정보를 파악하는 능력",
    "development_strategies":
        [
            "실생활 문제 제시로 동기 유발",
            "학생 간 질의응답 활동 통해 문제 조건 분석",
        ],
}
```

**예시 2**

```yaml
{
    "id": "CSE-REASONING-INDUCTIVE",
    "competencyId": "COMP-REASONING",
    "content": "구체적 예시 관찰·분석을 통해 일반적 결론을 이끌어내는 귀납적 추론 능력",
    "development_strategies":
        ["실험·관찰 과제 제시", "다양한 예시를 직접 만들어 보는 탐구 활동"],
}
```

---

## 5. 활용 시나리오

-   (Competency)-[:HAS_SUBELEMENT]->(CompetencySubElement) 혹은 (CompetencySubElement)-[:BELONGS_TO_COMPETENCY]->(Competency) 등의 관계로 상위 역량과 연결.
-   DB 질의 시, 한 교과 역량(예: “문제해결 역량”)에 어떤 하위 요소가 있고, 각각을 기르기 위한 교수전략은 무엇인지 쉽게 조회할 수 있음.
-   성취기준과 연결할 수도 있음. 예: (AchievementStandard)-[:EMPHASIZES_SUBELEMENT]->(CompetencySubElement)로 “이 성취기준은 귀납적 추론 하위 요소를 강조” 등 표현 가능.

---

### 요약

**CompetencySubElement** 노드는 **“수학 교과 역량(Competency)의 하위 요소”**를 저장하며, `content` 필드에 구체적 설명을 기입, `development_strategies`에 해당 요소를 함양하기 위한 교수·학습 방안을 기록합니다. 이를 (CompetencySubElement)-[:BELONGS_TO_COMPETENCY]->(Competency) 관계로 연결해, 역량을 한층 더 세밀하게 구조화·관리할 수 있습니다.

아래는 **데이터 스키마**에서 정의된 **2.3.3 TermSymbol** 노드에 대한 **데이타 설명서**(Data Dictionary) 예시입니다. 노드의 목적, 필드(컬럼), 타입, 제약조건, 필드 설명, 그리고 예시값 등을 종합하여 작성하였습니다.

---

# 2.3.3 TermSymbol 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: TermSymbol
-   **설명**:  
    수학 교과에서 사용하는 **용어(term)** 또는 **기호(symbol)** 정보를 저장하는 노드.
    -   예: `"약수"`, `"공배수"`, `"π"`, `"√"`, `"∠"`, 등
    -   해당 용어·기호가 처음 도입되는 학년군, 사용되는 맥락, 정의 등을 관리할 수 있다.
-   이를 통해 DB상에서 용어 · 기호에 대한 설명이나 최초 학습 시기를 구조적으로 파악하고, 교수 · 학습 설계에 참고 가능.

---

## 2. 필드(컬럼) 정의

| 필드명               | 자료형           | 제약조건         | 설명                                                      | 예시값                             |
| -------------------- | ---------------- | ---------------- | --------------------------------------------------------- | ---------------------------------- |
| **id**               | 문자열(고유, PK) | UNIQUE, NOT NULL | TermSymbol 노드의 고유 식별자                             | `"TS-PI"`                          |
| **type**             | 문자열           |                  | “용어(term)” 또는 “기호(symbol)”                          | `"기호(symbol)"`                   |
| **content**          | 문자열           | NOT NULL         | 해당 용어·기호 자체 또는 간단한 표현                      | `"π"`, `"약수"`, `"공배수"`, `"√"` |
| **gradeGroup**       | 문자열           |                  | 처음 도입되는 학년군(GradeGroup) id                       | `"GG-EL56"`                        |
| **first_introduced** | 문자열           |                  | 최초 등장 성취기준(AchievementStandard id) 혹은 관련 정보 | `"AS-5수01-02" (분수 개념 도입)"`  |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"TS-PI"`, `"TS-MINUS"`, `"TS-TERM-DIVISOR"`, 등
    - 용도: TermSymbol 노드를 고유하게 식별하기 위한 값.

2. **type**

    - 유형: 문자열
    - 예) `"기호(symbol)"`, `"용어(term)"`
    - 용도: 이 노드가 나타내는 것이 수학 “기호”인지, “용어”인지 구분.
    - 선택 사항으로, `type="symbol"` or `type="term"` 형태로도 가능.

3. **content**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"π"`, `"약수"`, `"공배수"`, `"∠"`, `"분수"`
    - 용도: 실제 기호나 용어를 기록한다.
    - 기호의 경우 그대로 `"√"`, `"π"` 등 입력, 용어의 경우 `"약수"`, `"최소공배수"` 등.

4. **gradeGroup**

    - 유형: 문자열
    - 예) `"GG-EL56"`, `"GG-MS13"`, `"GG-HIGH"`
    - 용도: 이 기호·용어가 처음 학습되거나 강조되는 학년군을 명시. (선택 입력)

5. **first_introduced**
    - 유형: 문자열
    - 예) `"AS-4수01-02"`(어느 성취기준에서 처음 등장), 또는 `"초등 4학년 교과서 3단원"`
    - 용도: 최초 도입되는 시점·맥락(성취기준 id 등)을 구체적으로 명시.

---

## 3. 제약(Constraints)

-   **id**: UNIQUE, NOT NULL
-   **content**: NOT NULL (기호나 용어 본체)
-   **type**, **gradeGroup**, **first_introduced**: 선택적으로 사용 가능.

---

## 4. 예시 레코드

**예시 1: 기호 π**

```yaml
{
    "id": "TS-PI",
    "type": "기호(symbol)",
    "content": "π",
    "gradeGroup": "GG-EL56",
    "first_introduced": "AS-5수03-15",
}
```

-   해설: π 기호는 초등 5~6학년군에서 원주율(AS-5수03-15) 학습 시 처음 도입된다는 정보.

**예시 2: 용어 “약수”**

```yaml
{
    "id": "TS-TERM-DIVISOR",
    "type": "용어(term)",
    "content": "약수",
    "gradeGroup": "GG-EL34",
    "first_introduced": "AS-4수01-04",
}
```

-   해설: “약수”는 초등 3~4학년군에서 도입되는 용어이며, 성취기준 `[4수01-04] 약수와 배수를 이해한다`에서 처음 등장.

**예시 3: 기호 √(루트)**

```yaml
{
    "id": "TS-ROOT",
    "type": "기호(symbol)",
    "content": "√",
    "gradeGroup": "GG-MS13",
    "first_introduced": "AS-9수01-07",
}
```

-   해설: 루트(√) 기호는 중학교(1~3학년군)에서 제곱근을 학습할 때 처음 소개된다는 예시.

---

## 5. 활용 시나리오

-   용어 · 기호가 어느 학년군(GradeGroup)에서, 어떤 성취기준(AchievementStandard)에서 최초 등장하는지 쉽게 파악.
-   (TermSymbol)-[:APPEARS_IN_STANDARD]->(AchievementStandard) 등 관계를 만들어도 됨. 혹은 `first_introduced` 필드만 사용해도 좋음.
-   DB 질의 시, “초등 5~6학년군에서 처음 배우는 기호 목록” 조회 가능, 또는 “AS-4수01-04에 등장하는 용어” 등 검색 가능.

---

### 요약

**TermSymbol** 노드는 **수학적 용어(term)**나 **기호(symbol)** 정보를 DB에 저장하여, 그 **등장 시점**(학년군, 성취기준)과 **용어·기호 자체**(`content`), **type**(“기호”/“용어”) 등을 관리합니다. 이를 통해 **교육과정 문서**, **교과서** 등에서 용어 · 기호를 언제·어떻게 도입하는지 추적하고, 현장에서 용어 · 기호 지도 시 참조할 수 있는 구조를 구축할 수 있습니다.

아래는 **데이터 스키마**에서 정의된 **2.4.1 TeachingMethod** 노드에 대한 **데이타 설명서**(Data Dictionary) 예시입니다. 노드의 목적, 필드(컬럼), 타입, 제약조건, 필드 설명, 그리고 예시값 등을 종합하여 작성하였습니다.

---

# 2.4.1 TeachingMethod 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: TeachingMethod
-   **설명**:  
    교육과정 및 수업 설계에서 활용하는 **교수법**(탐구학습, 협력학습, 프로젝트 학습, 모델링, 놀이·게임 학습 등)을 저장하는 노드.
    -   예: “탐구 학습”, “협력 학습”, “프로젝트 학습”, “모델링 수업”, “놀이 학습” 등
    -   주로 (Curriculum)-[:RECOMMENDS_METHOD]->(TeachingMethod)을 통해 “이 교육과정에서 권장하는 교수법”을 표현하거나, (EvaluationMethod), (TeachingStrategy) 등과 연계할 수도 있음.

---

## 2. 필드(컬럼) 정의

| 필드명                      | 자료형           | 제약조건         | 설명                                                                        | 예시값                                                                 |
| --------------------------- | ---------------- | ---------------- | --------------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| **id**                      | 문자열(고유, PK) | UNIQUE, NOT NULL | TeachingMethod 노드의 고유 식별자                                           | `"TM-PROJECT"`                                                         |
| **name**                    | 문자열           | NOT NULL         | 교수법 이름(“탐구 학습”, “협력 학습”, “프로젝트 학습” 등)                   | `"프로젝트 학습"`                                                      |
| **description**             | 문자열           |                  | 교수법에 대한 간략한 설명                                                   | `"학생 스스로 특정 과제를 탐구하고, 결과물을 발표·공유하는 학습 방안"` |
| **student_autonomy_level**  | 문자열           |                  | 학생 주도성 수준(“높음”, “중간”, “낮음”) 혹은 “교사 주도 vs 학생 주도” 구분 | `"학생 주도성 높음"`                                                   |
| **applicable_competencies** | 문자열 리스트    |                  | 적용 가능한 역량 목록(문제해결, 협력소통, 창의적 사고 등)                   | `["문제해결", "협력소통"]`                                             |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"TM-PROJECT"`, `"TM-INQUIRY"`, `"TM-COOP"`
    - 용도: TeachingMethod 노드를 식별하는 유일한 값.

2. **name**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"탐구 학습"`, `"협력 학습"`, `"프로젝트 학습"`, `"놀이·게임 학습"`
    - 용도: 교수법 이름.

3. **description**

    - 유형: 문자열
    - 예) `"학생이 문제나 과제를 스스로 탐색·연구하며 지식을 발견·구성하는 수업 방식"`
    - 용도: 해당 교수법에 대한 전반적 특징·설명.

4. **student_autonomy_level**

    - 유형: 문자열
    - 예) `"학생 주도성 높음"`, `"교사 주도성 중간"`
    - 용도: 이 교수법이 교사 중심인지, 학생 주도성이 높은지 등을 기술.

5. **applicable_competencies**
    - 유형: 문자열 리스트
    - 예) `["문제해결", "정보처리", "협력소통"]`
    - 용도: 이 교수법을 적용할 때 특히 중점이 되는 교과 역량(Competency) 목록.

-   **TeachingStrategy** 노드를 통해 구체적 차시나 수업 활동 설계를 기록하고, (TeachingMethod) 노드(예: “협력학습”)와 연계해 “이 전략은 협력학습 방법론 기반”이라고 표현할 수 있음.
-   **EvaluationTool**나 **AchievementStandard** 등과도 연계 가능: “이 성취기준(B수준)을 달성하려면 TSY-COOP-01 방식을 사용” 등.

---

### 요약

**TeachingStrategy** 노드는 **“구체적 수업 전략(활동, 자료, 차시 구성, 맞춤형 지도 등)”**을 저장합니다. `id`(고유 식별자)는 필수, 그 밖에 `targetLevel`, `strategyType`, `activities`, `materials`, `technology_usage` 등을 통해 실제 수업 장면에서 어떤 식으로 활동을 운영하는지 자세히 기술 가능합니다. 이를 **TeachingMethod**(더 넓은 범주)나 **AchievementStandard**와 연결하여, **교육과정** 운영의 실천적 정보를 DB에서 일관성 있게 관리할 수 있습니다.

# 2.4.3 EvaluationMethod 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: EvaluationMethod
-   **설명**:  
    수학 교수 · 학습에서 활용되는 **평가 방법**(지필평가, 프로젝트 평가, 토론평가, 포트폴리오 평가, 관찰·면담 등)을 저장하는 노드.
    -   예: “서술형평가”, “프로젝트평가”, “토론평가”, “포트폴리오평가” 등
    -   주로 (EvaluationMethod) 노드는 (Curriculum)과 연결돼 “교육과정에서 권장하는 평가 방법”을 나타내거나, (EvaluationTool) 노드(구체 문항세트)와 연계해서 “이 평가도구는 프로젝트 평가 방식으로 구성”이라고 표현 가능.

---

## 2. 필드(컬럼) 정의

| 필드명                    | 자료형           | 제약조건         | 설명                                                                      | 예시값                                      |
| ------------------------- | ---------------- | ---------------- | ------------------------------------------------------------------------- | ------------------------------------------- |
| **id**                    | 문자열(고유, PK) | UNIQUE, NOT NULL | EvaluationMethod 노드의 고유 식별자                                       | `"EM-PROJECT"`                              |
| **name**                  | 문자열           | NOT NULL         | 평가 방법 이름(“지필평가”, “프로젝트 평가”, “토론평가”, “서술형평가” 등)  | `"프로젝트 평가"`                           |
| **description**           | 문자열           |                  | 해당 평가 방법에 대한 간략한 설명                                         | `"프로젝트 과제 수행과 결과물을 종합 평가"` |
| **process_focused**       | 부울             |                  | 과정 중심 여부(true/false). 예: “프로젝트·포트폴리오 평가”는 과정 중심.   | `true`                                      |
| **suitable_competencies** | 문자열 리스트    |                  | 이 평가 방법이 중점적으로 확인·함양할 수 있는 역량(문제해결, 협력소통 등) | `["문제해결", "협력소통"]`                  |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"EM-PROJECT"`, `"EM-ESSAY"`, `"EM-DISCUSSION"`
    - 용도: EvaluationMethod 노드를 식별하는 유일한 값.

2. **name**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"프로젝트 평가"`, `"서술형 평가"`, `"포트폴리오 평가"`, `"관찰·면담 평가"`
    - 용도: 평가 방법(방식) 이름.

3. **description**

    - 유형: 문자열
    - 예) `"학생이 주도적으로 프로젝트를 수행하고 산출물·과정 등을 종합하여 평가"`
    - 용도: 해당 평가 방법의 전반적인 설명, 특징, 절차 요약.

4. **process_focused**

    - 유형: 부울
    - 예) `true` or `false`
    - 용도: 이 평가 방법이 과정 중심인지 여부. 예: 프로젝트·포트폴리오·토론 등은 대체로 과정 중심(`true`), 지필시험 등은 일반적으로 `false`.

5. **suitable_competencies**
    - 유형: 문자열 리스트
    - 예) `["창의적 사고", "협력소통"]`
    - 용도: 이 평가 방법을 활용하면 특히 잘 확인·함양할 수 있는 교과 역량(Competency)들을 기술.

-   **name**: NOT NULL
-   **description**, **process_focused**, **suitable_competencies**: 선택 입력 가능

# 2.4.4 EvaluationTool 데이터 설명서

### 필드별 상세 설명

    - 용도: EvaluationTool 노드를 식별하는 유일한 값.

2. **title**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"초3~4 도형 측정 A수준 지필평가"`, `"중학교 1~3학년 함수 프로젝트 평가"`

3. **itemType**

    - 유형: 문자열
    - 예) `"지필평가(객관식+단답형)"`, `"프로젝트(수행평가)"`, `"서술형평가"`

4. **targetLevel**

    - 유형: 문자열
    - 예) `"A"`, `"B"`, `"초등 3~4학년"`

    - 유형: 문자열

    - 예) `"낮음"`, `"중간"`, `"높음"`
    - 용도: 평가 도구 전체 난이도

7. **point**

    - 유형: 정수
    - 예) `30`, `40`, `100`
    - 용도: 총 배점(문항 전부 합한 점수)

8. **correctAnswer**

    - 유형: 문자열
    - 예) `"예시 답안 첨부"`
    - 용도: 전체 정답 안내(객관식 등), 또는 서술형 답안 예시.
    - 문항별 정답은 (AssessmentItem) 별로 관리 가능. 이 필드에는 전체 답안지 안내 등 큰 틀에서 기입할 수 있음.

9. **domain**

    - 유형: 문자열
    - 예) `"도형과 측정"`, `"확률과 통계"`
    - 용도: 어느 영역 평가인지(선택적으로 기입)

10. **gradeGroup**

    - 유형: 문자열
    - 예) `"1~2학년군"`, `"중1~3학년군"`
    - 용도: 대상 학년군 표시(선택).

11. **schoolLevel**

    - 유형: 문자열
    - 예) `"초등학교"`, `"중학교"`
    - 용도: 대상 학교급 표시(선택).

12. **evaluationFocus**

    - 유형: 문자열
    - 예) `"삼각형·사각형 구별과 넓이 계산"`, `"분수 사칙연산 이해"`
    - 용도: 평가가 집중하고자 하는 학습 요소·목표

13. **developmentPurpose**

    - 유형: 문자열
    - 예) `"단원 마무리 학습 점검"`, `"학기말 총괄평가"`, `"형성평가"`
    - 용도: 이 평가 도구를 제작·활용한 목적.

14. **considerationForUse**

    - 유형: 문자열
    - 예) `"계산기 사용 가능, 서술형 문항은 20분 내 작성"`
    - 용도: 활용 시 유의사항, 시험 시간·도구 사용 제한 등.

15. **conceptualUnderstanding**, **procedural_fluency**, **strategic_competence**, **adaptive_reasoning**, **productive_disposition**

    - 유형: 부울
    - 예) `true`/`false`
    - 용도: 이 평가도구에서 각 역량(개념 이해, 절차적 숙달, 전략적 역량, 적응적 추론, 생산적 태도)을 얼마나 평가하려는지 표시.

16. **online_applicable**
    - 유형: 부울
    - 예) `true`/`false`
    - 용도: 온라인 평가로 전환 가능 여부.

# 2.4.5 AssessmentItem 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: AssessmentItem
-   **설명**:  
    평가 도구(EvaluationTool)를 구성하는 **낱개 문항**(item)을 저장하는 노드.
    -   예: 객관식 문항, 단답형 문항, 서술형 문항, 논술형 문항 등
    -   (AssessmentItem) 노드들은 (EvaluationTool)-[:HAS_ASSESSMENT_ITEM]->(AssessmentItem) 관계로 연결되어, 특정 평가 도구 안에 포함된 문항을 구조적으로 관리함.

## 2. 필드(컬럼) 정의

| 필드명                     | 자료형           | 제약조건         | 설명                                                                   | 예시값                                                       |
| -------------------------- | ---------------- | ---------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------ |

| **content**                | 문자열           | NOT NULL         | 문항 내용(문제 텍스트, 상황 설명, 질문 등)                             | `"삼각형과 사각형 중 각각 몇 개를 만들 수 있는지 구하시오."` |
| **itemFormat**             | 문자열           |                  | 문항 형식(“객관식”, “단답형”, “서술형”, “논술형” 등)                   | `"객관식"`                                                   |
| **cognitiveLevel**         | 문자열           |                  | 인지 수준(“지식·이해”, “적용”, “분석”, “평가” 등 분류 시)              | `"적용"`                                                     |
| **scoreDistribution**      | 맵/JSON          |                  | 부분점수 배점 구조(예: 각 소문항 점수, 서술형 부분채점 등)             | `{"1번":2, "2번":3, "3번":5}`                                |
| **misconceptionDetection** | 부울             |                  | 오개념 탐지 여부(이 문항이 학생의 특정 오개념을 파악하는 데 유용한지)  | `true`                                                       |
| **itemSubtype**            | 문자열           |                  | 문항 세분류(“완성형”, “배합형”, “줄세우기” 등 필요 시 세부 분류)       | `"완성형 문항"`                                              |
| **evaluationElement**      | 문자열 리스트    |                  | 평가 요소(“도형 식별”, “넓이 계산 정확도”, “과정 서술 능력” 등) (복수) | `["도형 구분", "넓이 계산 과정"]`                            |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"AI-ITEM-001"`, `"AI-ITEM-002"`, `"AI-ITEM-003"`
    - 용도: AssessmentItem 노드를 식별하는 유일한 값.

2. **evaluationToolId**

    - 유형: 문자열
    - 예) `"ET-DOHYEONG-34A"`, `"ET-FRACTION-PROJECT"`
    - 용도: 해당 문항이 어느 평가 도구(EvaluationTool.id)에 포함되는지 참조.
    - 또한 (EvaluationTool)-[:HAS_ASSESSMENT_ITEM]->(AssessmentItem) 관계로도 표현 가능.

3. **content**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"삼각형과 사각형을 합쳐서 새로운 도형을 만들었을 때, 그 둘레를 구하시오."`
    - 용도: 실제 문항 텍스트, 문제 상황·질문 등.

4. **itemFormat**

    - 유형: 문자열
    - 예) `"객관식"`, `"단답형"`, `"서술형"`, `"논술형"`
    - 용도: 문제 형태를 요약.

5. **cognitiveLevel**

    - 유형: 문자열
    - 예) `"이해"`, `"적용"`, `"분석"`, `"평가"`
    - 용도: 문항이 요구하는 인지 수준(Bloom 분류 등)에 따라 표시.

6. **scoreDistribution**

    - 유형: 맵/JSON
    - 예) `{"(1)번 소문항":2, "(2)번 소문항":3, "(3)번 소문항":5}`
    - 용도: 부분점수 배점 구조나 서술형 부분채점 기준을 간단히 표현.

7. **misconceptionDetection**

    - 유형: 부울
    - 예) `true` or `false`
    - 용도: 이 문항이 특정 **오개념**(misconception) 발견에 유용한지 여부.

8. **itemSubtype**

    - 유형: 문자열
    - 예) `"완성형"`, `"배합형"`, `"줄세우기형"`, `"OX형"`, `"진위형"` 등
    - 용도: 문항을 더 세밀히 분류할 때 사용. (선택)

9. **evaluationElement**
    - 유형: 문자열 리스트
    - 예) `["도형 식별", "넓이 계산 과정 서술"]`
    - 용도: 이 문항이 주로 어떤 평가 요소(학습 요소)를 확인하는지(복수).

---

## 3. 제약(Constraints)

-   **id**: UNIQUE, NOT NULL
-   **content**: NOT NULL
-   나머지 필드는 선택적으로 입력 가능.

### 요약

# 2.4.6 LevelSpecificGuidance 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: LevelSpecificGuidance
-   **설명**:  
    특정 **평가 도구(EvaluationTool)**를 사용한 평가 결과로 나타난 **성취수준(A/B/C/D/E)**별 지도 지침을 저장하는 노드.
    -   예: 평가가 끝난 후 A수준 학생에게는 **심화 활동**을, C수준에게는 **보충 지도가** 필요하다는 지침을 서술하거나, 흔한 **오개념**을 교정하기 위한 **보충·심화 활동**을 제시.
    -   (LevelSpecificGuidance)은 (EvaluationTool)-[:HAS_LEVEL_GUIDANCE]->(LevelSpecificGuidance) 등으로 연결되어, 특정 평가도구 결과에서 A/B/C 각각 어떻게 지도할지 안내.

---

## 2. 필드(컬럼) 정의

| 필드명                   | 자료형           | 제약조건         | 설명                                              | 예시값                                                                |
| ------------------------ | ---------------- | ---------------- | ------------------------------------------------- | --------------------------------------------------------------------- |
| **id**                   | 문자열(고유, PK) | UNIQUE, NOT NULL | LevelSpecificGuidance 노드의 고유 식별자          | `"LSG-DOHYEONG-34A-C"`                                                |
| **evaluationToolId**     | 문자열           |                  | 어느 평가 도구(EvaluationTool.id)에 대한 지침인지 | `"ET-DOHYEONG-34A"`                                                   |
| **targetLevel**          | 문자열           |                  | 이 지침이 어느 성취수준(A,B,C,D,E)에 대한 것인지  | `"C"`                                                                 |
| **guidanceContent**      | 문자열           |                  | 지도 내용 전반에 대한 설명                        | `"도형 분류에서 실수한 학생을 위해 도형판을 활용한 반복 학습을 권장"` |
| **remedialActivities**   | 문자열 리스트    |                  | 보충 활동 목록                                    | `["도형판 활용 도형 구분 연습", "기초 넓이 계산 문제 제공"]`          |
| **enrichmentActivities** | 문자열 리스트    |                  | 심화 활동 목록                                    | `["복합도형 변형 문제 해결", "실생활 예시 찾아 발표"]`                |
| **commonMisconceptions** | 문자열 리스트    |                  | 해당 수준의 학생이 흔히 가질 수 있는 오개념       | `["삼각형 높이를 측정하지 못함", "직사각형 넓이 공식 혼동"]`          |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"LSG-DOHYEONG-34A-A"`, `"LSG-FRACTION-PROJECT-B"` 등
    - 용도: LevelSpecificGuidance 노드를 식별하는 유일한 값.

2. **evaluationToolId**

    - 유형: 문자열
    - 예) `"ET-DOHYEONG-34A"`, `"ET-FRACTION-PROJECT"`
    - 용도: 어느 평가도구와 연결되는 지침인지.
    - DB 관계로 (EvaluationTool)-[:HAS_LEVEL_GUIDANCE]->(LevelSpecificGuidance)로도 표현 가능.

3. **targetLevel**

    - 유형: 문자열
    - 예) `"A"`, `"B"`, `"C"` 등
    - 용도: 이 지침이 **어느 성취수준** 학생에게 제공되는지.

4. **guidanceContent**

    - 유형: 문자열
    - 예) `"도형 분류를 어려워한 학생에게는 기초 작도와 시각적 판별 활동을 추가 제공"`
    - 용도: 지도 내용 전반(글 형식).

5. **remedialActivities**

    - 유형: 문자열 리스트
    - 예) `["삼각형·사각형 도형판 활동", "직관적 분류 연습 문제"]`
    - 용도: 보충 지도에 필요한 구체 활동 목록.

6. **enrichmentActivities**

    - 유형: 문자열 리스트
    - 예) `["복합도형 변형 문제", "도형 면적 추론 프로젝트"]`
    - 용도: 심화 지도(우수 학생 등) 시 활용할 활동 목록.

7. **commonMisconceptions**
    - 유형: 문자열 리스트
    - 예) `["삼각형 높이 개념 혼동", "직사각형 변 길이 혼동"]`
    - 용도: 이 수준 학생들이 자주 나타내는 오개념 리스트.

-   **evaluationToolId**, **targetLevel**: 필요 시 NOT NULL로 설정(어느 평가도구, 어느 수준)
-   **guidanceContent**, **remedialActivities**, **enrichmentActivities**, **commonMisconceptions**: 선택 입력 가능

-   DB 질의로 “C수준 학생 지도방안”을 한눈에 조회, “A수준 학생 심화활동”도 별도로 조회.
-   교사가 성적 산출 후, 이 자료를 바탕으로 학생 개별 피드백·추가 학습 자료를 제공.

### 요약

# 2.4.7 ProcessAssessment 데이터 설명서

## 1. 노드(Node) 개요

-   **설명**:  
    **과정 중심 평가(프로젝트·포트폴리오·토론 등) 시, 평가의 진행 과정**(실행 지침, 피드백 전략 등)을 저장하는 노드.

---

## 2. 필드(컬럼) 정의

| 필드명                        | 자료형           | 제약조건         | 설명                                                                      | 예시값                                                     |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)

3. **focus_areas**

    - 예) `["협력소통", "정보처리", "문제해결 과정"]`

7. **studentReflectionPrompt**

    - 예) `"매 단계 마다 학생들이 학습일지를 작성, 자기평가 문항에 답하도록 함."`
    - 용도: 학생 자기 반성(Reflection) 유도 방안.

    -   예: “서술형 문항 채점 시 A는 개념·과정·정답을 완벽히 서술, B는 과정 일부 누락, …”

    - 용도: ScoringRubric 노드를 고유하게 식별.

    - 예) `{"A":"개념·과정 모두 정확", "B":"개념은 정확하나 과정 누락"}`

6. **exampleStudentResponse**

    - 용도: 수준별로 예시답안(“A 수준 학생이 쓴 모범답안”, “C 수준 답안” 등)을 저장해, 채점 가이드에 참고.

    - 예) `"초3"`, `"중1"`, `"고1"`
    - 용도: 어떤 학년을 대상으로 한 진단평가인지 명시.

3. **purpose**

    - 유형: 문자열
    - 예) `"기초학력 진단"`, `"학습 결손 파악"`, `"영역별 역량 점검"`
    - 용도: 평가를 실시하는 주요 목적.

4. **assessmentTime**

    - 유형: 문자열
    - 예) `"학년 초"`, `"1학기 초"`, `"분기별(학년 4분할)"`
    - 용도: 구체적 실시 시기.

5. **online_format**

    - 유형: 부울
    - 예) `true` or `false`
    - 용도: 온라인 평가 형식인지 여부.

6. **follow_up_strategies**

    - 유형: 문자열
    - 예) `"보충학습반 운영, 기초 미달학생 대상 1:1 지도 계획"`
    - 용도: 이 진단평가 결과를 활용하여 어떤 후속 조치를 할 것인지.

7. **checkMinimumStandard**
    - 유형: 부울
    - 예) `true` or `false`
    - 용도: 이 진단평가에서 **최소 성취수준 도달 여부**(기초학력 보장 등)를 중점으로 확인하는지.

## 2. 필드(컬럼) 정의

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"GS-MIDDLE-80CUTOFF"`, `"GS-HIGH-GRADE"` 등
    - 용도: GradingSystem 노드를 식별하는 유일한 값.

2. **name**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"중학교 성취평가(고정점수 컷오프)"`, `"고등학교 상대평가"`
    - 용도: 성적 산출 체계의 이름.

3. **gradeType**

    - 유형: 문자열
    - 예) `"중학교"`, `"고등학교"`, `"초등학교"`
    - 용도: 주로 어느 학교급에 적용하는지 표시.

4. **fixed_cutoff_method**

    - 유형: 부울
    - 예) `true` or `false`
    - 용도: 점수 분할이 고정형인지. 예) `true` ⇒ 90점 이상 A, 80~89 B, …

    - 유형: 문자열
    - 예) `"A:≥90점,B:80~89,C:70~79,D:60~69,E:<60"`
    - 용도: 각 등급(수준)별 점수 범위를 텍스트 형태로 저장.

# 2.5.2 LearningHistory 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: LearningHistory
-   **설명**:  
    학생이 특정 성취기준(AchievementStandard)을 학습한 과정과 결과(성취수준, 학습 상태 등)를 기록하는 노드.
    -   예: “학생 A가 [2수01-06] 성취기준을 학습 중이며, 달성 수준은 B”, “학생 B가 미적분Ⅰ 성취기준을 완료했으며, 최종 점수 85점” 등
    -   (Student)-[:HAS_LEARNING_HISTORY]->(LearningHistory)와 (LearningHistory)-[:RELATED_TO]->(AchievementStandard)로 구성하여, 학생별·성취기준별 학습 이력을 추적 가능.

---

## 2. 필드(컬럼) 정의

| 필드명              | 자료형              | 제약조건         | 설명                                                     | 예시값                                   |
| ------------------- | ------------------- | ---------------- | -------------------------------------------------------- | ---------------------------------------- |
| **id**              | 문자열(고유, PK)    | UNIQUE, NOT NULL | LearningHistory 노드의 고유 식별자                       | `"LH-000123"`                            |
| **studentId**       | 문자열              |                  | 어느 학생(Student.id)과 연결되는지                       | `"STU-2023001"`                          |
| **standardId**      | 문자열              |                  | 어느 성취기준(AchievementStandard.id)과 연결되는지       | `"AS-2수01-06"`                          |
| **status**          | 문자열              |                  | 학습 상태(“학습중”, “완료”, “미완료”, “학습보충필요” 등) | `"학습중"`                               |
| **achievedLevel**   | 문자열              |                  | 달성 수준(A,B,C,D,E 등)                                  | `"B"`                                    |
| **strengths**       | 문자열 리스트       |                  | 학습 강점                                                | `["연산 계산 속도 빠름"]`                |
| **weaknesses**      | 문자열 리스트       |                  | 학습 약점                                                | `["서술 과정 미흡", "자릿값 개념 혼동"]` |
| **completion_date** | 문자열              |                  | 학습 완료 일자                                           | `"2023-05-10"`                           |
| **learning_path**   | 문자열(혹은 리스트) |                  | 학습 경로(예: “기초→보충→심화” 등), 또는 단계별 자료     | `"보충→기초문제→심화문제"`               |
| **finalScore**      | 정수                |                  | 최종 점수(시험 등)                                       | `85`                                     |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"LH-000123"`, `"LH-STU2023-45"`
    - 용도: LearningHistory 노드를 식별하는 유일한 값.

2. **studentId**

    - 유형: 문자열
    - 예) `"STU-2023001"`
    - 용도: 어느 학생과 연결되는지. 보통 (Student)-[:HAS_LEARNING_HISTORY]->(LearningHistory) 관계에서 참조.

3. **standardId**

    - 유형: 문자열
    - 예) `"AS-2수01-06"`
    - 용도: 어느 성취기준에 대한 학습 이력인지. (LearningHistory)-[:RELATED_TO]->(AchievementStandard)로도 표현 가능.

4. **status**

    - 유형: 문자열
    - 예) `"학습중"`, `"완료"`, `"미완료"`, `"보충필요"`
    - 용도: 현재 학습 상태를 나타냄.

5. **achievedLevel**

    - 유형: 문자열
    - 예) `"A"`, `"B"`, `"C"`
    - 용도: 학생이 해당 성취기준에서 최종적으로 도달한 성취수준.

6. **strengths**

    - 유형: 문자열 리스트
    - 예) `["연산이 빠름", "개념 정리 능숙"]`
    - 용도: 학습 과정에서 파악한 학생 강점.

7. **weaknesses**

    - 유형: 문자열 리스트
    - 예) `["실수 잦음", "문제 해결 과정 서술 미흡"]`
    - 용도: 학습 과정에서 파악한 학생 약점.

8. **completion_date**

    - 유형: 문자열 (날짜 형식)
    - 예) `"2023-05-10"`
    - 용도: 학습 완료일, 또는 평가 마무리 일자.

9. **learning_path**

    - 유형: 문자열(혹은 배열)
    - 예) `"보충단계→기초문제→심화문제"`
    - 용도: 어떤 학습 단계를 거쳤는지 기록.

10. **finalScore**
    - 유형: 정수
    - 예) `85`
    - 용도: 시험 결과 점수 등 수치로 표현된 최종 기록(선택).

## 3. 제약(Constraints)

-   **studentId**, **standardId**: 필요 시 NOT NULL로 설정(이력이 어떤 학생, 어떤 성취기준과 관련된 것인지 식별).

## 5. 활용 시나리오

-   (Student)-[:HAS_LEARNING_HISTORY]->(LearningHistory) : 한 학생에게 여러 학습 이력이 연결.
-   (LearningHistory)-[:RELATED_TO]->(AchievementStandard) : 해당 이력이 어떤 성취기준 학습과 관련 있는지 표현.

**LearningHistory**는 **“학생별·성취기준별 학습 이력”**을 관리하는 노드로, `studentId`, `standardId`, `status`, `achievedLevel`, `strengths`, `weaknesses`, `completion_date`, `finalScore` 등을 통해 학습 과정과 결과를 상세히 기록합니다. 이를 통해 **학생 개별 학습추적**, **학습 결손 파악**, **보충·심화 지도** 등을 DB에서 일관되게 지원할 수 있습니다.

# 2.5.3 PersonalizedRecommendation 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: PersonalizedRecommendation
-   **설명**:  
    **개별 학생 맞춤형 추천**(학습 경로, 보충·심화 내용, 성취기준 재학습 등)을 저장하는 노드.
    -   예: “학생 A에게 분수연산 심화자료 추천”, “학생 B에게 도형 개념 보충 지침 발행”
    -   (Student)-[:RECEIVES_RECOMMENDATION]->(PersonalizedRecommendation) 관계를 통해, 한 학생에게 발행된 여러 학습 추천을 저장할 수 있음.

---

## 2. 필드(컬럼) 정의

| 필드명                  | 자료형           | 제약조건         | 설명                                                      | 예시값                              |
| ----------------------- | ---------------- | ---------------- | --------------------------------------------------------- | ----------------------------------- |
| **id**                  | 문자열(고유, PK) | UNIQUE, NOT NULL | PersonalizedRecommendation 노드의 고유 식별자             | `"PR-000123"`                       |
| **studentId**           | 문자열           |                  | 어느 학생(Student.id)에게 발행된 추천인지                 | `"STU-2023001"`                     |
| **recommendedStandard** | 문자열           |                  | 추천 대상 성취기준(AchievementStandard id), 필요 시 명시  | `"AS-2수01-06"`                     |
| **reason**              | 문자열           |                  | 추천 이유(“분수 곱셈 보충 필요”, “도형 심화 학습권장” 등) | `"최근 테스트 결과 분수 연산 취약"` |
| **recommendation_type** | 문자열           |                  | 추천 유형(“보충”, “심화”, “재학습”, “추가 프로젝트” 등)   | `"보충"`                            |
| **ai_generated**        | 부울             |                  | AI 자동 생성 여부(true면 AI가 생성, false면 교사 수동)    | `true`                              |
| **priority_level**      | 문자열           |                  | 우선순위(“높음·중간·낮음” 등)                             | `"높음"`                            |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"PR-000123"`, `"PR-STU2023-02"`
    - 용도: PersonalizedRecommendation 노드를 고유하게 식별.

2. **studentId**

    - 유형: 문자열
    - 예) `"STU-2023001"`
    - 용도: 어느 학생에게 발행된 추천인지.
    - (Student)-[:RECEIVES_RECOMMENDATION]->(PersonalizedRecommendation) 관계로도 표현 가능.

3. **recommendedStandard**

    - 유형: 문자열
    - 예) `"AS-2수01-06"`, `"AS-9수02-04"`
    - 용도: 어떤 성취기준 학습을 보충/심화 추천하는지. (선택)

4. **reason**

    - 유형: 문자열
    - 예) `"최근 분수 곱셈 문항에서 오류 다수 발견"`, `"도형 작도 단계 생략 문제 잦음"`
    - 용도: 교사나 AI가 추천을 발행한 근거.

5. **recommendation_type**

    - 유형: 문자열
    - 예) `"보충"`, `"심화"`, `"재학습"`, `"추가 프로젝트"`, `"AI 튜터링 안내"`
    - 용도: 추천의 종류를 한 단어로 표현.

6. **ai_generated**

    - 유형: 부울
    - 예) `true` or `false`
    - 용도: AI 자동 생성(추천 시스템)인지, 교사 수동 발행인지.

7. **priority_level**
    - 유형: 문자열
    - 예) `"높음"`, `"중간"`, `"낮음"`
    - 용도: 추천의 중요도·우선순위.

---

## 3. 제약(Constraints)

-   **id**: UNIQUE, NOT NULL
-   나머지 필드(studentId, recommendedStandard, reason, recommendation_type, ai_generated, priority_level)는 필요 시 선택적으로 입력.

---

## 5. 활용 시나리오

-   (Student)-[:RECEIVES_RECOMMENDATION]->(PersonalizedRecommendation) : 한 학생이 여러 맞춤형 추천(보충·심화)을 발행받을 수 있음.
-   DB 질의 “AI가 자동 생성했고 우선순위가 높음인 추천 목록”을 필터링해 교사가 우선 조치.
-   “recommendation_type=보충”인 경우 보충학습 자료나 수업을 연결, “심화”면 확장 과제 안내.

### 요약

**PersonalizedRecommendation** 노드는 **“학생 개별 맞춤형 학습 추천”** 정보를 저장합니다. `studentId`로 어느 학생에 대한 추천인지, `recommendedStandard`로 어느 성취기준 학습과 관련있는지, `reason`과 `recommendation_type`으로 왜·무엇을 추천하는지 표현합니다. `ai_generated`, `priority_level` 등을 통해 **AI 자동화 여부**와 **우선순위**도 관리하여, **학습 지원 시스템**이 적절한 시점에 학생에게 알림하고, 교사도 DB에서 손쉽게 확인할 수 있게 합니다.

# 2.5.4 BasicCompetency 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: BasicCompetency
-   **설명**:  
    **기초학력(최소 성취기준 등) 관련 정보**를 저장하는 노드.
    -   예: “초등 1~2학년군에서 최소 성취기준으로 설정된 성취기준 목록”, “책임교육 학년에서 지원 전략” 등
    -   (BasicCompetency) 노드는 기초학력이 보장되어야 할 학년이나 학생들에게 제공될 **지원 전략**(supportStrategies), **특별 지원 조치**(special_support_measures) 등을 DB에서 구조적으로 관리하기 위한 목적.

---

## 2. 필드(컬럼) 정의

| 필드명                       | 자료형           | 제약조건         | 설명                                                                       | 예시값                                            |
| ---------------------------- | ---------------- | ---------------- | -------------------------------------------------------------------------- | ------------------------------------------------- |
| **id**                       | 문자열(고유, PK) | UNIQUE, NOT NULL | BasicCompetency 노드의 고유 식별자                                         | `"BC-EL12-001"`                                   |
| **minimumStandards**         | 문자열 리스트    |                  | 최소 성취기준 목록(복수). 특정 학년에서 꼭 도달해야 할 성취기준 id나 내용  | `["AS-2수01-06", "AS-2수02-03"]`                  |
| **supportStrategies**        | 문자열 리스트    |                  | 기초학력 보장 위해 제공할 지원 전략(보충 지도, 맞춤형 학습 자료 등) (복수) | `["기초연산 교재 제공", "방과후 보충수업 편성"]`  |
| **isResponsibleGrade**       | 부울             |                  | 책임교육학년제 적용 여부(true이면 필수 이수 학년 등)                       | `true`                                            |
| **responsibleGradeType**     | 문자열           |                  | 책임교육학년 유형(“중1”, “초3” 등), 필요 시 명시                           | `"초3 책임교육학년"`                              |
| **special_support_measures** | 문자열           |                  | 특별 지원 조치(예: “매주 1회 교사 1:1 코칭”, “상시 튜터 운영”)             | `"매주 1회 기초학력 코칭, 지자체 지원 튜터 운영"` |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"BC-EL12-001"`, `"BC-MIDDLE-002"`
    - 용도: BasicCompetency 노드를 식별하는 유일한 값.

2. **minimumStandards**

    - 유형: 문자열 리스트
    - 예) `["AS-2수01-06", "AS-도형-01"]`
    - 용도: 해당 학년·학습단계에서 **반드시 도달해야 할 최소 성취기준** 목록(id).

3. **supportStrategies**

    - 유형: 문자열 리스트
    - 예) `["기초연산 학습지 제공", "방과후 교실 운영", "1:1 튜터링 지원"]`
    - 용도: 기초학력(최소 성취기준)을 달성 못 한 학생을 위해 제공할 **지원 전략**(복수).

4. **isResponsibleGrade**

    - 유형: 부울
    - 예) `true`/`false`
    - 용도: 이 노드가 “책임교육학년”인지 여부. (책임교육학년제: 해당 학년에 기초학력을 보장해야 하는 제도)

5. **responsibleGradeType**

    - 유형: 문자열
    - 예) `"초3"`, `"중1"`
    - 용도: 책임교육학년의 구체적인 학년(혹은 학년군). `isResponsibleGrade=true`일 때 세부 명시 가능.

6. **special_support_measures**
    - 유형: 문자열
    - 예) `"학습부진 학생 대상 주 2회 특별지도, 시·도 지원강사 배치"`
    - 용도: 특수·추가 지원 방안이 있을 경우(행정기관 지원, 추가 예산 투입 등).

---

## 3. 제약(Constraints)

-   **id**: UNIQUE, NOT NULL
-   나머지 필드(minimumStandards, supportStrategies, isResponsibleGrade, responsibleGradeType, special_support_measures)는 필요에 따라 입력.

### 요약

**BasicCompetency** 노드는 **“기초학력(최소 성취기준) 및 지원 전략”**을 다루는 노드로, `minimumStandards`에 기초학력 보장에 필요한 성취기준을 배열로 저장하고, `supportStrategies`와 `special_support_measures`로 **학습부진 예방 및 보충**을 위한 구체 방안을 기록합니다. 이를 통해 **책임교육학년** 여부(`isResponsibleGrade`)도 함께 관리하여, **교육과정 운영**에서 **취약학생 지원** 체계를 DB에서 일관되게 구축할 수 있습니다.

# 2.6.1 TeachingResource 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: TeachingResource
-   **설명**:  
    수업에 활용되는 교수 자료(예: PPT, 동영상, 워크시트, 참고문헌 등)를 저장하는 노드.
    -   예: “삼각형 넓이 계산 PPT”, “도형 학습 워크시트 PDF”, “확률 개념 소개 동영상” 등
    -   (TeachingStrategy)-[:USES_RESOURCE]->(TeachingResource) 등 관계를 통해, 특정 수업 전략이나 차시 활동에서 어떤 자료를 사용하는지 연결 가능.

---

## 2. 필드(컬럼) 정의

| **resourceType**        | 문자열           |                  | 자료 유형(“PPT”, “동영상”, “워크시트”, “참고문헌” 등)   | `"PPT"`                                           |
| **digital_format**      | 부울             |                  | 디지털 형식 여부(true면 디지털 파일, false면 인쇄물 등) | `true`                                            |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"TR-0001"`, `"TR-VIDEO-101"`
    - 용도: TeachingResource 노드를 식별하는 고유 값.

2. **title**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"삼각형 넓이 학습 PPT"`, `"분수 연산 실습 워크시트"`
    - 용도: 자료 제목.

3. **resourceType**

    - 유형: 문자열
    - 예) `"PPT"`, `"동영상"`, `"워크시트"`, `"참고서적"`, `"PDF자료"`
    - 용도: 자료 유형.

4. **digital_format**

    - 유형: 부울
    - 예) `true` (디지털 파일) or `false` (인쇄된 교재 등)
    - 용도: 해당 자료가 디지털인지 아닌지 여부.

5. **online_accessible**

6. **description**

    - 유형: 문자열
    - 예) `"삼각형 넓이 공식과 예시를 PPT 슬라이드로 정리"`
    - 용도: 자료 요약 설명.

7. **educational_purpose**

    - 예) `"삼각형 넓이 개념 도입용"`, `"분수 연산 실습"`

-   나머지 필드(resourceType, digital_format, online_accessible, description, educational_purpose): 필요 시 선택적 입력

# 2.6.2 DigitalTool 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: DigitalTool
-   **설명**:  
    수학 수업·학습에서 활용할 수 있는 **디지털 도구**(그래프 앱, CAS, 온라인 플랫폼, 시뮬레이션 앱 등)를 저장하는 노드.
    -   예: “GeoGebra”, “Desmos”, “WolframAlpha CAS”, “온라인 러닝 플랫폼(LMS)” 등
    -   (TeachingStrategy)-[:USES_DIGITAL_TOOL]->(DigitalTool) 과 같은 관계를 통해, 특정 수업전략이 어떤 디지털도구를 사용해 수업을 진행하는지 나타낼 수 있음.

---

## 2. 필드(컬럼) 정의

| 필드명                     | 자료형           | 제약조건         | 설명                                                         | 예시값                                |
| -------------------------- | ---------------- | ---------------- | ------------------------------------------------------------ | ------------------------------------- |
| **id**                     | 문자열(고유, PK) | UNIQUE, NOT NULL | DigitalTool 노드의 고유 식별자                               | `"DT-0001"`                           |
| **name**                   | 문자열           | NOT NULL         | 디지털 도구 이름(“GeoGebra”, “Desmos”, “Python” 등)          | `"GeoGebra"`                          |
| **toolType**               | 문자열           |                  | 도구 유형(“그래프 앱”, “CAS”, “통계분석 SW”, “LMS” 등)       | `"그래프 앱"`                         |
| **platform**               | 문자열           |                  | 플랫폼(“웹”, “Windows”, “Android”, “iOS” 등)                 | `"웹/모바일"`                         |
| **accessibility_level**    | 문자열           |                  | 접근성 수준(“낮음”, “중간”, “높음”) 혹은 “모두 사용 가능” 등 | `"높음"`                              |
| **offline_available**      | 부울             |                  | 오프라인 사용 가능 여부                                      | `true`                                |
| **supported_competencies** | 문자열 리스트    |                  | 지원되는 역량(“문제해결”, “추론”, “정보처리” 등)             | `["정보처리", "문제해결"]`            |
| **technical_requirements** | 문자열           |                  | 기술 요구사항(“Java 설치 필요”, “Chrome 최신 버전 권장” 등)  | `"Java 8 이상, Chrome 최신버전 권장"` |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 예) `"DT-0001"`, `"DT-GEOGEBRA"`, `"DT-DESMOS"`
    - 용도: DigitalTool 노드를 식별하는 유일한 값.

2. **name**

    - 유형: 문자열
    - 제약: NOT NULL
    - 예) `"GeoGebra"`, `"Desmos"`, `"WolframAlpha"`
    - 용도: 디지털 도구의 이름.

3. **toolType**

    - 유형: 문자열
    - 예) `"그래프 앱"`, `"CAS"`, `"3D 기하 도구"`, `"LMS"`, `"통계분석 SW"`
    - 용도: 도구의 기능적 유형.

4. **platform**

    - 유형: 문자열
    - 예) `"웹 기반"`, `"Windows 앱"`, `"iOS/Android 지원"`
    - 용도: 지원 플랫폼.

5. **accessibility_level**

    - 유형: 문자열
    - 예) `"높음"`, `"중간"`, `"낮음"`
    - 용도: 접근성(사용 용이성, 장애인 접근성 등)을 서술할 수 있음.

6. **offline_available**

    - 유형: 부울
    - 예) `true`/`false`
    - 용도: 인터넷 없이도 사용 가능 여부.

7. **supported_competencies**

    - 유형: 문자열 리스트
    - 예) `["문제해결", "연결", "정보처리"]`
    - 용도: 이 도구를 사용하면 **특히 강화되거나 유용한 역량**을 기술.

8. **technical_requirements**
    - 유형: 문자열
    - 예) `"Java 8 필요", "Chrome 최신버전", "512MB 이상 메모리"`
    - 용도: 도구 실행을 위한 기술적인 전제 조건.

### 요약

**DigitalTool** 노드는 **“교육 현장에 활용되는 디지털 도구”**(그래프 앱, CAS, 온라인 플랫폼 등)를 저장하며, `id`와 `name`이 핵심이며, `toolType`, `platform`, `accessibility_level`, `offline_available`, `supported_competencies`, `technical_requirements` 등을 통해 도구의 특성과 요구사항을 구체적으로 관리합니다. 이를 **TeachingStrategy** 등과 연결해, **수업 설계**나 **학생 활동**에서 어떤 디지털도구를 사용할지 DB에서 일관성 있게 운영할 수 있습니다.

아래는 **데이터 스키마**에서 정의된 **2.6.3 OnlineLearningEnvironment** 노드에 대한 **데이타 설명서**(Data Dictionary) 예시입니다. 노드의 목적, 필드(컬럼), 타입(자료형), 제약조건, 그리고 예시값 등을 종합하여 작성하였습니다.

# 2.6.4 MisconceptionInfo 데이터 설명서

## 1. 노드(Node) 개요

-   **노드명(Label)**: MisconceptionInfo
-   **설명**:  
    수학 학습에서 발생하는 **오개념(흔한 오류, 잘못된 이해)** 정보를 저장하는 노드.
    -   예: “분수 곱셈에서 분자끼리만 곱하고 분모는 유지하는 오개념”, “피타고라스 정리에서 빗변을 잘못 이해” 등
    -   각 오개념에 대해 **교정 전략(remedyStrategies)**, **관련 성취기준(relatedStandards)** 등을 함께 기록하여 교사·학습자에게 참고자료를 제공.

---

## 2. 필드(컬럼) 정의

| 필드명               | 자료형           | 제약조건         | 설명                                                                                 | 예시값                                                     |
| -------------------- | ---------------- | ---------------- | ------------------------------------------------------------------------------------ | ---------------------------------------------------------- |
| **id**               | 문자열(고유, PK) | UNIQUE, NOT NULL | MisconceptionInfo 노드의 고유 식별자                                                 | `"MI-0001"`                                                |
| **topic**            | 문자열           |                  | 오개념이 주로 발생하는 주제(“분수 곱셈”, “도형 작도”, “일차방정식” 등)               | `"분수 곱셈"`                                              |
| **description**      | 문자열           |                  | 오개념 설명(어떻게 잘못 이해하는지, 어떤 부분이 오류인지)                            | `"분수 곱셈에서 분모는 무시하고 분자끼리만 곱한다고 생각"` |
| **remedyStrategies** | 문자열 리스트    |                  | 해당 오개념을 교정하기 위한 전략(“시각적 모델 제시”, “단계별 예시 문제”) (복수 가능) | `["시각적 분수 모델을 활용", "동수누가 개념 재확인"]`      |
| **relatedStandards** | 문자열 리스트    |                  | 이 오개념이 관련된 성취기준 목록                                                     | `["AS-5수01-09", "AS-5수01-10"]`                           |

### 필드별 상세 설명

1. **id**

    - 유형: 문자열(고유, PK)
    - 제약: UNIQUE, NOT NULL
    - 예) `"MI-0001"`, `"MI-FRACTION-02"`
    - 용도: MisconceptionInfo 노드를 식별하는 유일한 값.

2. **topic**

    - 유형: 문자열
    - 예) `"분수 곱셈"`, `"도형 작도"`, `"일차함수 해석"`
    - 용도: 오개념이 주로 나타나는 수학적 주제(단원)·영역을 표시.

3. **description**

    - 유형: 문자열
    - 예) `"학생들이 분수 곱셈에서 분모는 그대로 둬야 한다고 착각함"`
    - 용도: 구체적으로 어떤 잘못된 개념이나 오류인지 서술.

4. **remedyStrategies**

    - 유형: 문자열 리스트
    - 예) `["실물 모델로 분수 곱셈 시각화", "동수누가 관계 강조"]`
    - 용도: 이 오개념을 교정·완화하기 위한 수업 전략, 학습 자료 활용, 예시 문제 등.

5. **relatedStandards**
    - 유형: 문자열 리스트
    - 예) `["AS-5수01-09", "AS-5수01-10"]`
    - 용도: 관련되는 성취기준 id 목록.
    - 이로써 어떤 성취기준 학습 중에 이 오개념이 자주 발견되는지 알 수 있음.

---

## 3. 제약(Constraints)

-   **id**: UNIQUE, NOT NULL
-   나머지 필드(topic, description, remedyStrategies, relatedStandards)는 필요 시 선택적으로 입력

## 5. 활용 시나리오

-   (MisconceptionInfo) 노드를 통해, 수학 교육에서 발생하는 오개념과 그 교정전략을 DB에 축적.
-   수업 설계 시 교사가 (AchievementStandard)와 (MisconceptionInfo) 연결을 참조해, “이 성취기준을 지도할 때 자주 나타나는 오류와 해소방안”을 사전에 대비.
-   DB 질의: “분수 관련 오개념” or “도형 작도 관련 오개념” 목록만 필터해 일괄 검색 가능.

---

### 요약

**MisconceptionInfo** 노드는 **“수학 학습 과정에서 빈번히 나타나는 오개념(흔한 오류)”**를 기록하고, 해당 오개념을 **설명(description)**, **교정 방안(remedyStrategies)**, 그리고 어떤 성취기준과 연관되는지(relatedStandards) 등을 저장합니다. 이를 통해 교사·연구자들이 수업·교재 개발 시 자주 발생하는 오류를 예측하고, 효과적인 **교정 전략**을 미리 준비할 수 있도록 지원합니다.
