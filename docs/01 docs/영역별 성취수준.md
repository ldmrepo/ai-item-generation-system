### **설명**

- `domainAchievementLevel.domainId`: 영역 ID(예: 수와 연산 = D01, 변화와 관계 = D02).
- `domainAchievementLevel.domainName`: 해당 영역의 이름(수와 연산, 변화와 관계 등).
- `levels`: 각 단계(A~E)별로 **지식·이해**(knowledgeUnderstanding), **과정·기능**(processSkill), **가치·태도**(valueAttitude)를 기술.
- 원문에서 **A, B, C, D, E** 각각에 대하여 문서에 언급된 내용(“지식·이해”, “과정·기능”, “가치·태도”)을 **그대로 옮기거나**, 문서 표현을 **적절히 요약**해 JSON 구조에 맞추었습니다.
- 문서에서 공란으로 제시된 일부 영역(또는 항목)은 `""`(빈 문자열)로 처리하거나, 해당 key 자체를 생략할 수 있습니다.

위와 같은 형태로 **영역별 성취수준** 정보를 JSON 구조로 추출함으로써, 시스템적으로 **성취수준 관리**, **교수·학습 설계**, **맞춤형 평가** 등을 체계적으로 수행할 수 있습니다.
